# Изменение размеров панелей в редакторе форм

## Текущее состояние

- **Редактор форм**: [FormPreviewApp.tsx](src/webview/components/FormPreview/FormPreviewApp.tsx) — EDT-подобный UI с тремя зонами:
  - **Левая панель** (`edt-pane--left`): вкладки «Элементы» / «Командный интерфейс», дерево и поиск.
  - **Правая панель** (`edt-pane--right`): вкладки «Свойства» / «Реквизиты» / «Команды» / «Параметры».
  - **Нижняя зона** (`edt-bottom`): заголовок «Дизайнер» и канвас.
- **Вёрстка**: [editor.css](src/webview/styles/editor.css) (строки 1835–2105):
  - `.edt-layout` — grid с двумя строками: `minmax(220px, 1fr)` и `minmax(180px, 0.9fr)` (фиксированные пропорции).
  - `.edt-top` — grid с двумя колонками: `minmax(260px, 34%)` и `1fr` (левая/правая панель).
- В проекте **нет** библиотек для resizable-панелей (только React/DOM). Resize в других местах — через CSS `resize: vertical` или разовые решения (например, col-resize в DcsEditor).

## Целевое поведение

- Пользователь может перетаскивать:
  1. **Вертикальный разделитель** между панелью «Элементы» (слева) и «Реквизиты» (справа) — меняется ширина левой/правой панели.
  2. **Горизонтальный разделитель** между верхним блоком (Элементы + Реквизиты) и панелью «Дизайнер» — меняется высота верхней и нижней зоны.
- Ограничения: минимальные ширины/высоты панелей, чтобы контент оставался usable (например, левая панель не уже ~200px, нижняя не ниже ~120px).

## Варианты реализации


| Подход                                                             | Плюсы                                                         | Минусы                                            |
| ------------------------------------------------------------------ | ------------------------------------------------------------- | ------------------------------------------------- |
| **A. Своя логика resize** (useState + mousedown/mousemove/mouseup) | Без новых зависимостей, полный контроль, небольшой объём кода | Нужно самому обрабатывать мышь, подписки, min/max |
| **B. Библиотека** (например `react-resizable-panels`)              | Готовый UX, доступность, коллапс панелей при желании          | Новая зависимость, возможная избыточность         |


Рекомендация: **вариант A** — минимальные изменения, без новых зависимостей, в духе правил проекта.

## План реализации

### 1. Состояние размеров в FormPreviewApp

- Добавить state для размеров, заданных в процентах (или px) от контейнера:
  - `leftPaneWidthPercent` (например, по умолчанию 34, min 20, max 60).
  - `bottomPanelHeightPercent` (например, по умолчанию 47, min 15, max 75).
- Размеры можно хранить только в рамках сессии (при перезагрузке webview — сброс к умолчаниям). Сохранение в `localStorage` или через `vscode.getState()` — опционально, в плане не обязательно.

### 2. Вертикальный разделитель (левая/правая панель)

- Между `edt-pane--left` и `edt-pane--right` вставить элемент-разделитель (например `<div className="edt-resizer edt-resizer--vertical" />`).
- Заменить grid в `.edt-top` на flex (или оставить grid с явными `fr`/процентами из state): левая панель — `width: leftPaneWidthPercent%`, правая — `flex: 1` (или оставшаяся доля).
- Обработка мыши на разделителе:
  - `onMouseDown` — запомнить начальную позицию мыши и текущие проценты, установить флаг «идёт перетаскивание», подписаться на `document` `mousemove` и `mouseup`.
  - В `mousemove` — пересчитывать `leftPaneWidthPercent` от позиции мыши относительно `.edt-top` (clientWidth), с учётом min/max.
  - В `mouseup` — снять подписки, сбросить флаг.
- Курсор на разделителе: `cursor: col-resize`; при перетаскивании можно добавить класс на `body` и глобальный `user-select: none`, чтобы не выделялся текст.

### 3. Горизонтальный разделитель (верх / Дизайнер)

- Между `.edt-top` и `.edt-bottom` вставить горизонтальный разделитель (`edt-resizer--horizontal`).
- Заменить grid в `.edt-layout` на flex (или две строки с явной высотой): верх — `flex: 1` с `min-height` и явной высотой из state (например через `flex-basis` в % от контейнера), низ — оставшаяся доля; либо задать высоту нижней панели в % и верхней `flex: 1`.
- Проще всего: контейнер `edt-layout` — `display: flex; flex-direction: column; height: 100%`. Верхний блок с `flex: 1`, `min-height: 220px`, высота задаётся не через fr, а через state — например хранить `bottomPanelHeightPercent` и считать высоту нижней панели как `height: bottomPanelHeightPercent%` от `edt-layout`, верхней — `flex: 1 min-height: 220px`. Альтернатива — хранить одну долю (например высоту низа в %) и один разделитель.
- Обработка мыши аналогично: `cursor: row-resize`, расчёт от `clientY` и высоты `.edt-layout`.

### 4. Стили в editor.css

- Классы для разделителей:
  - `.edt-resizer` — базовая полоса (например 4–6px), фон/граница, курсор.
  - `.edt-resizer--vertical` — между панелями, `cursor: col-resize`, по центру между левой и правой панелью.
  - `.edt-resizer--horizontal` — между верхом и низом, `cursor: row-resize`.
- При необходимости: зона захвата чуть шире визуальной полоски (увеличить hit area) для удобства перетаскивания.
- Адаптировать существующие правила `.edt-top` и `.edt-layout`: убрать жёсткие `grid-template-rows/columns`, задать размеры через inline-стили из React (width/height в % или flex) так, чтобы разделители не «ломали» layout.

### 5. Файлы для изменения

- [src/webview/components/FormPreview/FormPreviewApp.tsx](src/webview/components/FormPreview/FormPreviewApp.tsx) — state для размеров, разметка двух разделителей, обработчики мыши (можно вынести в кастомные хуки `useVerticalResize` и `useHorizontalResize` в том же файле или в отдельном утильном файле, если хочется переиспользовать).
- [src/webview/styles/editor.css](src/webview/styles/editor.css) — новые классы `.edt-resizer`, `.edt-resizer--vertical`, `.edt-resizer--horizontal`; правки `.edt-layout` и `.edt-top` под управляемые размеры (flex или grid с явными размерами из расчёта).

## Риски и ограничения

- При очень маленьком webview минимальные размеры могут занимать большую долю — оставить разумные min (например левая панель ≥ 200px, нижняя ≥ 120px в px при маленькой высоте можно не увеличивать сверх 100%).
- Если позже добавить сохранение размеров (localStorage/vscode state), нужно будет передавать размеры при инициализации и сохранять при изменении.

## Дополнительные идеи (не в объёме задачи)

- Сохранение размеров панелей между сессиями (localStorage или `webviewPanel.webview.getState`).
- Двойной клик по разделителю — сброс к размерам по умолчанию.
- Использование библиотеки `react-resizable-panels` при желании единообразного UX и возможного коллапса панелей в будущем.
