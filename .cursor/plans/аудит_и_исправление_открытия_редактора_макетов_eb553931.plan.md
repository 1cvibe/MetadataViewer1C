#Аудит и исправление открытия редактора макетов

## Проблемы

1. **Неправильное определение корня конфигурации**: `rootPath` устанавливается как `workspaceFolders[0].uri`, но `element.id` содержит абсолютный путь к корню конфигурации. Они могут не совпадать.
2. **Некорректная нормализация путей**: Путь формируется как `${element.id}/Documents/...`, но затем неправильно обрабатывается в `openTemplate`.
3. **Проблемы с регистром и разделителями**: Пути в ошибках показывают `d:/1C/BASE` (строчные буквы, прямые слэши), а реальные пути - `D:\1C\BASE` (заглавные буквы, обратные слэши).
4. **Неполная логика удаления element.id**: Текущая логика пытается определить `element.id` по паттерну, но не учитывает, что `element.id` - это абсолютный путь к корню конфигурации.

## Решение

### 1. Исправить определение корня конфигурации в `openTemplate`

**Файл**: [src/metadataView.ts](src/metadataView.ts)**Проблема**: В методе `openTemplate` (строка 196) используется `this.rootPath`, который может не совпадать с корнем конфигурации, указанным в `element.id`.**Решение**:

- Если путь `template` начинается с абсолютного пути (определяется по наличию диска на Windows или `/` на Unix), использовать его как корень
- Если путь относительный, использовать `this.rootPath`
- Нормализовать пути: конвертировать в абсолютные, использовать правильные разделители для текущей платформы

### 2. Улучшить нормализацию путей

**Файл**: [src/metadataView.ts](src/metadataView.ts)**Изменения в методе `openTemplate`**:

- Определить, является ли `template` абсолютным путем
- Если абсолютный - использовать его напрямую, убрав только `Ext/Template` если есть
- Если относительный - использовать `rootPath` как базу
- Нормализовать разделители путей для текущей платформы
- Использовать `path.resolve` для корректной обработки путей

### 3. Исправить формирование пути в `CreateTreeElements`

**Файл**: [src/metadataView.ts](src/metadataView.ts)**Проблема**: На строке 1368 путь формируется как `${element.id}/${CreatePath(objectName)}/Templates/...`, что создает смешанный путь с абсолютным началом и относительными сегментами.**Решение**:

- Использовать `pathJoin` вместо конкатенации строк
- Убедиться, что путь формируется корректно для текущей платформы
