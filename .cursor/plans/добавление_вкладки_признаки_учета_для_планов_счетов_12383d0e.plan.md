# План: Добавление вкладки "Признаки учета" для планов счетов

## Анализ текущей структуры

### Структура данных AccountingFlag и ExtDimensionAccountingFlag в XML

Элементы `AccountingFlag` и `ExtDimensionAccountingFlag` находятся в `ChildObjects` плана счетов и имеют структуру:

```xml
<AccountingFlag uuid="...">
    <Properties>
        <Name>Количественный</Name>
        <Type>
            <v8:Type>xs:boolean</v8:Type>
        </Type>
        <!-- другие свойства -->
    </Properties>
</AccountingFlag>
<ExtDimensionAccountingFlag uuid="...">
    <Properties>
        <Name>Суммовой</Name>
        <Type>
            <v8:Type>xs:boolean</v8:Type>
        </Type>
        <!-- другие свойства -->
    </Properties>
</ExtDimensionAccountingFlag>
```

### Текущая обработка

- **Парсинг**: В `src/xmlParsers/metadataParser.ts` элементы `AccountingFlag` и `ExtDimensionAccountingFlag` не парсятся
- **Сохранение**: В `src/utils/xmlDomUtils.ts` функция `applyChildObjectsChangesWithDom` обрабатывает только `Attribute`, `Resource`, `Dimension` и `TabularSection`
- **Определение типа объекта**: В `src/xmlParsers/metadataParser.ts` для `ChartOfAccounts` тип остается как "ChartOfAccounts" (не переводится в `objectTypeMap`)

## Изменения

### 1. Обновление интерфейса ParsedMetadataObject

**Файл**: `src/xmlParsers/metadataParser.ts`

Добавить опциональные поля для хранения признаков учета:

```typescript
export interface ParsedMetadataObject {

    accountingFlags?: ParsedAttribute[]; // Признаки учета по счетам

}
```

### 2. Добавление парсинга AccountingFlag и ExtDimensionAccountingFlag

**Файл**: `src/xmlParsers/metadataParser.ts`

- Создать функцию `parseAccountingFlags(childObjects: any): ParsedAttribute[]` для парсинга `AccountingFlag`
- Создать функцию `parseExtDimensionAccountingFlags(childObjects: any): ParsedAttribute[]` для парсинга `ExtDimensionAccountingFlag`
- В функции `parseMetadataXml` добавить парсинг признаков учета только для объектов типа `ChartOfAccounts`:
```typescript
const parsed = {
    // ... существующие поля ...
    accountingFlags: objectType === 'ChartOfAccounts' 
        ? parseAccountingFlags(objNode.ChildObjects) 
        : undefined,
    extDimensionAccountingFlags: objectType === 'ChartOfAccounts'
        ? parseExtDimensionAccountingFlags(objNode.ChildObjects)
        : undefined
};
```

## Детали реализации

### Определение типа объекта "план счетов"

Использовать проверку:

```typescript
const isChartOfAccounts = 
  selectedObject?.objectType === 'ChartOfAccounts' ||
  (selectedObject?.sourcePath && selectedObject.sourcePath.includes('ChartsOfAccounts'));
```

### Структура данных для признаков учета

При парсинге XML структура:

- `AccountingFlag` и `ExtDimensionAccountingFlag` находятся в `ChildObjects`
- Имеют структуру, аналогичную `Attribute` с `Properties` внутри
- Тип всегда `xs:boolean`

### Валидация типа

При добавлении/редактировании признака учета:

- Тип должен быть строго `xs:boolean`
- Если пользователь пытается изменить тип, показывать предупреждение или блокировать изменение
- В модальном окне TypeWidget должен быть заблокирован или показывать только boolean
