## План реализации

### 1. Добавление настроек в package.json

В файле [`package.json`](package.json) в секции `contributes.configuration[0].properties` добавить две новые настройки:

- **`metadataViewer.mcpServerName`** (тип: `string`, по умолчанию: `"1c-ai"`)
  - Описание: "Имя mcp сервера 1C-напарник"
  - Используется для определения имени сервера в `~/.cursor/mcp.json`

- **`metadataViewer.codeGenerationToolName`** (тип: `string`, по умолчанию: `"ask_1c_ai"`)
  - Описание: "Имя инструмента для генерации кода"
  - Содержит только часть имени инструмента после префикса `mcp_<server>_`

### 2. Обновление MCPClient.ts

В файле [`src/autogen-bsl/generators/MCPClient.ts`](src/autogen-bsl/generators/MCPClient.ts):

#### 2.1. Обновление метода `ask1cAi()`

- Заменить жестко закодированное имя инструмента `'mcp_1c-ai_ask_1c_ai'` на динамическое формирование
- Читать настройки через `vscode.workspace.getConfiguration('metadataViewer')`
- Формировать полное имя инструмента как `mcp_${serverName}_${toolName}`, где:
  - `serverName` берется из настройки `metadataViewer.mcpServerName`
  - `toolName` берется из настройки `metadataViewer.codeGenerationToolName`

#### 2.2. Обновление метода `resolveServerName()`

- Использовать настройку `metadataViewer.mcpServerName` вместо жестко закодированного `'1c-ai'`
- Обновить логику определения сервера для поддержки настраиваемого имени сервера

#### 2.3. Обновление метода `mapToolName()`

- Использовать настройку `metadataViewer.mcpServerName` для формирования префикса
- Обновить логику удаления префикса для поддержки настраиваемого имени сервера

### 3. Структура изменений

```typescript
// В ask1cAi():
const config = vscode.workspace.getConfiguration('metadataViewer');
const serverName = config.get<string>('mcpServerName', '1c-ai');
const toolName = config.get<string>('codeGenerationToolName', 'ask_1c_ai');
const fullToolName = `mcp_${serverName}_${toolName}`;
return await this.callMCPTool(fullToolName, { ... });
```

### 4. Обратная совместимость

- Значения по умолчанию совпадают с текущими жестко закодированными значениями
- Существующий код продолжит работать без изменений настроек
