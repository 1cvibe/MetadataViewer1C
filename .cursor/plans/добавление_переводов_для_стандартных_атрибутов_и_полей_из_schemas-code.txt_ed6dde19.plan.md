# План добавления переводов для стандартных атрибутов и полей из schemas-code.txt

## Цель

Добавить переводы для всех полей, которые используются в `schemas-code.txt` и определены в схемах, но отсутствуют в словаре переводов `FIELD_LABELS` в файле `src/Metadata/field-values.ts`.

## Результаты полного анализа schemas-code.txt

Проведен полный анализ файла `scripts/schemas-code.txt`:

- Всего полей из properties: 165
- Всего полей из title: 196
- Всего полей с префиксами (xr:, v8:, cfg:, app:): 27
- Всего уникальных полей: 197
- Существующих переводов: 214

**Результат:** После фильтрации значений enum и служебных полей, отсутствуют переводы для **14 полей**:

- 13 стандартных атрибутов
- 1 поле с префиксом (TypeReductionMode)

## Обнаруженная проблема

В файле `src/schemas/baseSchema.ts` определены 16 стандартных атрибутов в enum свойства `name` схемы `standardAttributeSchema`:

```typescript
enum: ['Posted', 'Ref', 'DeletionMark', 'Date', 'Number', 'Owner', 'Parent', 'Code', 'Description', 'Predefined', 'Level', 'IsFolder', 'DataVersion', 'LockedByDBMS', 'WriteMode', 'ReadOnly']
```

Из них только 3 имеют переводы в `FIELD_LABELS`:

- `Description` - 'Описание'
- `Predefined` - 'Предопределенное'
- `WriteMode` - 'Режим записи'

**Отсутствуют переводы для 14 полей:**

### Стандартные атрибуты (13):

1. `Posted` - Проведен
2. `Ref` - Ссылка
3. `DeletionMark` - Пометка удаления
4. `Date` - Дата
5. `Number` - Номер
6. `Owner` - Владелец
7. `Parent` - Родитель
8. `Code` - Код
9. `Level` - Уровень
10. `IsFolder` - Это папка
11. `DataVersion` - Версия данных
12. `LockedByDBMS` - Заблокировано СУБД
13. `ReadOnly` - Только чтение

### Другие поля (1):

14. `TypeReductionMode` - Режим приведения типа (используется как `xr:TypeReductionMode` в схемах, но функция `getFieldLabel()` убирает префикс перед поиском перевода)

## Использование в коде

- Все 16 стандартных атрибутов используются в `scripts/schemas-code.txt`
- `StandardAttributes` используется в 19 местах в `src/schemas/objectSchemas.ts`
- Стандартные атрибуты отображаются в UI через функцию `getFieldLabel()` в `src/webview/components/FormEditor.tsx` (строки 1256-1269), которая использует `FIELD_LABELS` для перевода имен атрибутов
- Имена стандартных атрибутов берутся из свойства `name` объекта стандартного атрибута и должны переводиться через `getFieldLabel()`

## Структура добавляемых переводов

Каждый перевод добавляется в формате:

```typescript
'AttributeName': 'Перевод на русский язык',
```

Переводы должны быть:

- Понятными и точными
- Соответствующими терминологии 1С:Предприятие
- Согласованными с существующими переводами

## Дополнительные проблемы, обнаруженные на скриншоте

При просмотре интерфейса обнаружены дополнительные поля без переводов:

1. **Проблема с отображением переводов в FormEditor.tsx:**

   - На строках 1280, 1310, 1341 используется `{key}:` напрямую без перевода через `getFieldLabel()`
   - Нужно заменить на `{getFieldLabel(key)}:`

2. **Отсутствующие переводы для полей стандартных атрибутов:**

   - `PredefinedDataName` - Имя предопределенных данных
   - `Order` - Порядок
   - `OffBalance` - Внебалансовый

## Ожидаемый результат

После выполнения плана все 16 стандартных атрибутов и все поля из `schemas-code.txt` будут иметь переводы в `FIELD_LABELS`, что обеспечит корректное отображение названий полей в интерфейсе редактора метаданных. Функция `getFieldLabel()` автоматически убирает префиксы (xr:, v8: и т.д.) перед поиском перевода, поэтому для полей с префиксами нужно добавлять переводы без префикса.

Также будут исправлены места в `FormEditor.tsx`, где названия полей отображаются без перевода, и добавлены переводы для всех полей, видимых в интерфейсе.
