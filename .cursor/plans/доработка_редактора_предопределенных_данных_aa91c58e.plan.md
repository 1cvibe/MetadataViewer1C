# Доработка редактора предопределенных данных

## 1. Переработка UI дизайна

### Текущее состояние

- Редактор использует простую таблицу с инлайн-редактированием
- UI находится в `src/predefinedDataPanel.ts` (строки 650-869) - встроенный React компонент в HTML строке

### Целевое состояние

- Использовать карточки (cards) как во вкладке "Реквизиты" редактора метаданных
- Стили из `src/webview/styles/editor.css` (классы `.attribute-card`, `.attribute-header`, `.attribute-properties`)
- Пример реализации: `src/webview/components/FormEditor.tsx` (строки 1037-1143)

### Изменения

- Заменить таблицу на grid с карточками в `src/predefinedDataPanel.ts`
- Использовать CSS классы из `editor.css` для единообразия
- Каждая карточка содержит: имя, код, наименование, тип (если доступен), папка, действия

## 2. Ограничение поля "Тип" только для планов видов характеристик

### Текущее состояние

- Поле "Тип" доступно для всех объектов метаданных
- Определяется в `src/predefinedDataPanel.ts` (строка 763)

### Изменения

- Проверять тип объекта метаданных при инициализации панели
- Скрывать/показывать поле "Тип" в зависимости от `objectType === 'ChartOfCharacteristicTypes'`
- Логика проверки аналогична `src/webview/components/MetadataEditor.tsx` (строки 176-180)

## 3. Правильное заполнение Code для справочников

### Текущее состояние

- Code сохраняется как строка в `src/xmlParsers/predefinedSerializer.ts`
- Функция `isNumericCode()` определяет, является ли Code числом (строки 16-24)
- Для новых элементов добавляется `xsi:type="xs:decimal"` если Code числовой (строка 416)

### Проблема

- При обновлении существующих элементов может теряться формат Code
- Нужно сохранять исходный формат (xsi:type) для существующих элементов

### Изменения

- В `src/xmlParsers/predefinedSerializer.ts` функция `updateItemElementInDom()` уже сохраняет xsi:type (строки 150-163)
- Убедиться, что при создании новых элементов для справочников Code правильно форматируется
- Для справочников (Catalog) проверять, является ли Code числом, и добавлять xsi:type="xs:decimal"

## 4. Правильное сохранение структуры XML

### Текущее состояние

- Используется `serializePredefinedXmlWithDom()` в `src/xmlParsers/predefinedSerializer.ts`
- Сохраняется `originalXml` в `src/xmlParsers/predefinedParser.ts` (строка 146)
- Механизм сохранения через xmldom (согласно памяти `xml-parsing-standard-xmldom`)

### Изменения

- Убедиться, что при добавлении/изменении/удалении используется `serializePredefinedXmlWithDom()`
- Сохранять структуру XML (элементы vs атрибуты, порядок, форматирование)
- Добавлять BOM при сохранении (уже реализовано в строке 454-458)

## 5. Файлы для изменения

- `src/predefinedDataPanel.ts` - основной файл редактора (UI и логика)
- `src/xmlParsers/predefinedSerializer.ts` - сериализация XML (возможно, доработки для Code)
- `src/webview/styles/editor.css` - стили (если нужно добавить специфичные для предопределенных)

## 6. Особенности реализации

### UI компонент

- Переписать React компонент в `getHtmlForWebview()` (строки 673-826)
- Использовать карточки вместо таблицы
- Модальные окна для добавления/редактирования (аналогично FormEditor)

### Проверка типа объекта

- В `PredefinedDataPanel.createOrShow()` уже определяется `objectType` (строки 88-100)
- Передать `objectType` в webview через сообщение `init`
- В React компоненте проверять `objectType === 'ChartOfCharacteristicTypes'` для показа поля "Тип"

### Code для справочников

- При создании нового элемента проверять `objectType === 'Catalog'`
- Если Code числовой, добавлять `xsi:type="xs:decimal"`
- Использовать функцию `isNumericCode()` из serializer
