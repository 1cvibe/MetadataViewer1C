# План доработки редактора предопределенных элементов для плана счетов

## Обзор задачи

Добавить специальные поля для объектов типа "План счетов" в редактор предопределенных данных. Новые поля должны быть видимы и доступны только для объектов типа `ChartOfAccounts`.

## Структура изменений

### 1. Расширение интерфейсов данных

**Файл:** `src/predefinedDataInterfaces.ts`

- Расширить интерфейс `PredefinedDataItem` новыми опциональными полями:
  - `Parent?: string` - родительский элемент (имя родителя, только для отображения, readonly). Если элемент находится на верхнем уровне (не в ChildItems), родитель - технический "Счета" (не отображается в XML)
  - `AccountType?: 'Active' | 'Passive' | 'ActivePassive'` - вид счета
  - `OffBalance?: boolean` - забалансовый счет
  - `Order?: string` - порядок
  - `AccountingFlags?: Array<{ flagName: string; enabled: boolean }>` - признаки учета
  - `ExtDimensionTypes?: Array<{ dimensionType: string; turnoverOnly: boolean; sum: boolean; flags: Record<string, boolean> }>` - виды субконто

### 2. Создание модуля для работы с данными плана счетов

**Новый файл:** `src/metadata_utils/ChartOfAccountsDataLoader.ts`

Модуль для загрузки данных из XML объекта плана счетов:

- `loadChartOfAccountsMetadata(configRoot: string, objectName: string)` - загружает XML объекта плана счетов
- `getAccountingFlags(metadataXml: string)` - извлекает список признаков учета (`<AccountingFlag>`)
- `getExtDimensionAccountingFlags(metadataXml: string)` - извлекает список признаков учета по субконто (`<ExtDimensionAccountingFlag>`)
- `getExtDimensionTypes(metadataXml: string)` - извлекает виды субконто (`<ExtDimensionTypes>`) с ссылками на план видов характеристик
- `loadPredefinedDimensionTypes(configRoot: string, chartOfCharacteristicTypesName: string)` - загружает предопределенные элементы из плана видов характеристик

### 3. Расширение парсера Predefined.xml

**Файл:** `src/xmlParsers/predefinedParser.ts`

- Расширить функцию `parseItemElement()` для парсинга новых полей:
  - `<AccountType>` → `AccountType` (Active, Passive, ActivePassive)
  - `<OffBalance>` → `OffBalance` (boolean)
  - `<Order>` → `Order` (string)
  - `<AccountingFlags>` → `AccountingFlags` (массив объектов)
  - `<ExtDimensionTypes>` → `ExtDimensionTypes` (массив объектов)
- Обработать иерархическую структуру через `ChildItems` (уже реализовано)
- **Важно:** Поле `Parent` не парсится из XML, а вычисляется динамически:
  - При рекурсивном парсинге `ChildItems` передавать имя родительского элемента
  - Если элемент на верхнем уровне, устанавливать `Parent = "Счета"`
  - Если элемент внутри `ChildItems`, устанавливать `Parent = имя родителя`

### 4. Расширение сериализатора Predefined.xml

**Файл:** `src/xmlParsers/predefinedSerializer.ts`

**Принципы сохранения XML структуры (из памяти проекта):**

- Использовать `@xmldom/xmldom` (DOMParser, XMLSerializer) - уже используется
- Сохранять исходную структуру XML через DOM манипуляции
- Применять `formatXml()` из `xmlDomUtils.ts` для форматирования
- Сохранять BOM при сохранении файла
- Сохранять порядок элементов и форматирование исходного XML

**Расширение функции `updateItemElementInDom()`:**

- Добавление/обновление простых элементов:
  - `<AccountType>` → значение: Active, Passive, ActivePassive
  - `<OffBalance>` → значение: true/false (boolean)
  - `<Order>` → строковое значение
- Создание/обновление таблицы `<AccountingFlags>`:
  - Структура: `<AccountingFlags><Item><FlagName>...</FlagName><Enabled>...</Enabled></Item>...</AccountingFlags>`
  - Если элемент существует - обновить содержимое, сохраняя структуру
  - Если элемента нет - создать новый с правильной структурой
  - Сохранять порядок элементов Item внутри AccountingFlags
- Создание/обновление таблицы `<ExtDimensionTypes>`:
  - Структура: `<ExtDimensionTypes><Item><DimensionType>...</DimensionType><Turnover>...</Turnover><Sum>...</Sum><Flags>...</Flags></Item>...</ExtDimensionTypes>`
  - Вложенная структура `<Flags>` с элементами `<Item><FlagName>...</FlagName><Enabled>...</Enabled></Item>`
  - Сохранять порядок элементов и вложенную структуру
- Сохранение иерархии `ChildItems`:
  - Рекурсивная обработка вложенных элементов через `ChildItems`
  - Сохранять структуру вложенности при обновлении
- **Важно:** Поле `Parent` не сохраняется в XML, так как оно вычисляется из иерархической структуры `ChildItems`

**Расширение функции `createItemElementFromTemplate()`:**

- При создании нового элемента добавлять новые поля для плана счетов:
  - Создавать элементы `<AccountType>`, `<OffBalance>`, `<Order>` если они указаны
  - Создавать структуру `<AccountingFlags>` с элементами `<Item>` если указаны признаки учета
  - Создавать структуру `<ExtDimensionTypes>` с вложенными `<Flags>` если указаны виды субконто
- Использовать шаблон из исходного XML для сохранения форматирования

**Файл:** `src/predefinedDataPanel.ts`

- В методе `postInitialData()` загружать дополнительные данные для плана счетов:

### 6. Расширение React компонента редактора

**Файл:** `src/webview/components/PredefinedEditor/PredefinedEditorApp.tsx`

- Добавить проверку типа объекта: `isChartOfAccounts = objectType === 'ChartOfAccounts'`
- Добавить состояние для данных плана счетов из `chartOfAccountsData`
- Расширить форму добавления/редактирования элемента:
  - **Родитель** - текстовое поле только для чтения (readonly), отображает имя родительского элемента. Логика определения:
    - Если элемент находится внутри `ChildItems` другого элемента, родитель - имя этого элемента
    - Если элемент находится на верхнем уровне (корневой элемент в `PredefinedData`), родитель - "Счета" (технический уровень, не отображается в XML)

  - **Забалансовый** - checkbox

  - **Виды субконто** - таблица с колонками:
    - "Вид субконто" (выпадающий список из предопределенных элементов плана видов характеристик)
    - "Только обороты" (checkbox)
    - "Суммовой" (checkbox)
    - Динамические колонки для каждого признака учета по субконто (checkbox)
- Создать компоненты для таблиц:
  - `AccountingFlagsTable` - таблица признаков учета
  - `ExtDimensionTypesTable` - таблица видов субконто

### 7. Обработка иерархии элементов

- При парсинге `Predefined.xml` учитывать иерархию через `ChildItems`
- При определении родителя элемента:
  - Если элемент находится внутри `ChildItems` другого элемента, `Parent` = имя родительского элемента
  - Если элемент находится на верхнем уровне (корневой уровень в `PredefinedData`), `Parent` = "Счета" (технический уровень)
- При отображении в редакторе показывать иерархическую структуру (отступы, группировка)
- При сохранении сохранять структуру `ChildItems` в XML:
  - Рекурсивно обрабатывать вложенные элементы через `ChildItems`
  - Сохранять порядок элементов на каждом уровне иерархии
  - При обновлении элемента сохранять его `ChildItems` с правильной структурой
  - При создании нового элемента с `ChildItems` создавать правильную вложенную структуру
- Поле "Родитель" вычисляется автоматически на основе позиции элемента в иерархии, не сохраняется в XML

### 8. Правильная сборка XML структуры при сохранении

**Ключевые принципы (из памяти проекта и существующих планов):**

1. **Использование xmldom:**

   - Всегда использовать `@xmldom/xmldom` (DOMParser, XMLSerializer)
   - Работать с DOM структурой для сохранения исходной структуры XML
   - Не использовать `fast-xml-parser` для сохранения

2. **Сохранение исходной структуры:**

   - Использовать `originalXml` из `parsePredefinedXmlWithDom()` как основу
   - При обновлении существующих элементов сохранять их исходную структуру (порядок элементов, форматирование)
   - При создании новых элементов использовать шаблон из исходного XML

3. **Обработка новых полей для плана счетов:**

   - При добавлении `<AccountType>`, `<OffBalance>`, `<Order>` - вставлять в правильном порядке (после `Description`, перед `ChildItems`)
   - При создании `<AccountingFlags>` - создавать правильную структуру с элементами `<Item>`
   - При создании `<ExtDimensionTypes>` - создавать правильную структуру с вложенными `<Flags>`
   - Сохранять порядок элементов внутри таблиц (AccountingFlags, ExtDimensionTypes)

4. **Форматирование XML:**

   - Применять `formatXml()` из `xmlDomUtils.ts` после сериализации
   - Сохранять отступы и переносы строк
   - Сохранять BOM в начале файла

5. **Обработка иерархии:**

   - При обновлении элемента сохранять его `ChildItems` рекурсивно
   - При создании нового элемента с `ChildItems` создавать правильную вложенную структуру
   - Сохранять порядок элементов на каждом уровне иерархии

6. **Тестирование сохранения:**

   - Проверить сохранение структуры XML (элементы vs атрибуты)
   - Проверить сохранение форматирования (отступы, переносы строк)
   - Проверить сохранение BOM
   - Проверить обработку иерархии `ChildItems`
   - Проверить добавление/обновление/удаление элементов с новыми полями

## Технические детали

### Парсинг XML структуры

Пример структуры для элемента плана счетов в `Predefined.xml`:

```xml
<Item>
  <Name>Товары</Name>
  <Code>01.02</Code>
  <Description>Товары</Description>
  <AccountType>Active</AccountType>
  <OffBalance>false</OffBalance>
  <Order>01.02</Order>
  <AccountingFlags>
    <Item>
      <FlagName>Количественный</FlagName>
      <Enabled>true</Enabled>
    </Item>
  </AccountingFlags>
  <ExtDimensionTypes>
    <Item>
      <DimensionType>Номенклатура</DimensionType>
      <Turnover>false</Turnover>
      <Sum>true</Sum>
      <Flags>
        <Item>
          <FlagName>Количественный</FlagName>
          <Enabled>true</Enabled>
        </Item>
      </Flags>
    </Item>
  </ExtDimensionTypes>
  <ChildItems>
    <!-- вложенные элементы -->
  </ChildItems>
</Item>
```

**Примечание:** Поле `Parent` не сохраняется в XML, а вычисляется динамически на основе позиции элемента в иерархии `ChildItems`. Если элемент находится на верхнем уровне (корневой уровень в `PredefinedData`), родитель считается техническим "Счета".

### Получение данных из XML объекта плана счетов

- Парсить `ChartsOfAccounts/Управленческий/Управленческий.xml`
- Извлекать из `ChildObjects`:
  - `<AccountingFlag>` → список имен признаков учета
  - `<ExtDimensionAccountingFlag>` → список имен признаков учета по субконто
  - `<ExtDimensionTypes>` → список видов субконто с ссылками на план видов характеристик

### Получение предопределенных элементов плана видов характеристик

- Использовать `MetadataScanner` для поиска объекта плана видов характеристик
- Парсить `Ext/Predefined.xml` для получения списка предопределенных элементов
- Кэшировать результаты для производительности
