#Исправление отображения макета при открытии из XML

## Проблемы

1. Применяется автомасштаб через `transform: scale()`, что искажает реальные размеры
2. Ширина колонок не учитывается из `document.columns[].columnsItem[].column.formatIndex`
3. Высота строк не учитывается из формата строк (`document.format[].height`)
4. Границы ячеек отображаются только как простые (1px solid), без учета типа линии, толщины и цвета
5. Нужно проверить применение всех стилей шрифта

## Решения

### 1. Убрать автомасштаб

**Файл:** `src/webview/components/TemplateEditor/TemplateTable.tsx`

- Убрать или сделать опциональным `transform: scale(${zoom})` в контейнере таблицы (строка ~366)
- Убрать связанные стили `width` и `height` с расчетом от zoom
- Оставить zoom только для масштабирования содержимого, но не применять к контейнеру таблицы

### 2. Применить ширину колонок

**Файл:** `src/webview/components/TemplateEditor/TemplateTable.tsx`

- В заголовках колонок (строка ~409-419) вместо фиксированного `minWidth: '100px'` использовать ширину из формата колонки
- Создать функцию `getColumnWidth(col: number): string | undefined`, которая:
- Находит колонку в `templateDocument.columns[].columnsItem[]` по индексу
- Получает `formatIndex` из `column.formatIndex`
- Возвращает `width` из `templateDocument.format[formatIndex - 1].width`
- Применить ширину через `style={{ width: columnWidth || '100px' }}` в `<th>` заголовков колонок
- Также применить ширину к ячейкам таблицы через `colgroup` или стили колонок

### 3. Применить высоту строк

**Файл:** `src/webview/components/TemplateEditor/TemplateTable.tsx`

- В строках таблицы (строка ~441) применить высоту из формата строки
- Создать функцию `getRowHeight(rowIndex: number): string | undefined`, которая:
- Находит строку в `templateDocument.rowsItem[]` по индексу
- Получает `formatIndex` из `row.row.formatIndex`
- Возвращает `height` из `templateDocument.format[formatIndex - 1].height`
- Применить высоту через `style={{ height: rowHeight || 'auto' }}` в `<tr>`

### 4. Улучшить отображение границ

**Файл:** `src/webview/components/TemplateEditor/TemplateTable.tsx`

- Проверить, есть ли в XML данные о типе линии, толщине и цвете границ (возможно в `document.line[]`)
- Если данные есть, добавить их в интерфейс `TemplateFormat` в `src/templatInterfaces.ts`:
- `leftBorderLineType?: string` (сплошная, точечная, двойная, редкий пунктир, частый пунктир, большой пунктир)
- `leftBorderWidth?: number` (толщина)
- `leftBorderColor?: string` (цвет)
- Аналогично для `topBorder`, `bottomBorder`, `rightBorder`
- Создать функцию `getBorderStyle(borderType: number, lineType?: string, width?: number, color?: string): string`, которая:
- Преобразует тип линии в CSS `border-style` (solid, dotted, dashed, double)
- Использует толщину из параметра или значение по умолчанию
- Использует цвет из параметра или значение по умолчанию
- В коде применения границ (строки ~538-554) использовать эту функцию вместо простого `'1px solid'`

### 5. Проверить стили шрифта

**Файл:** `src/webview/components/TemplateEditor/TemplateTable.tsx`

- Убедиться, что все свойства шрифта применяются (строки ~558-584):
- `fontFamily` - применяется
- `fontSize` - применяется
- `fontWeight` (bold) - применяется
- `fontStyle` (italic) - применяется
- `textDecoration` (underline, strikeout) - применяется
- `transform: scale()` для масштаба шрифта - применяется
- Если какие-то свойства не применяются, добавить их

### 6. Обновить CSS для корректного отображения

**Файл:** `src/webview/components/TemplateEditor/template-editor.css`

- Убедиться, что таблица использует `table-layout: fixed` для корректного применения ширины колонок
- Убрать или скорректировать стили, которые могут влиять на масштабирование
