# Исправление проблем редактора макетов (часть 2)

## Обзор

План включает исправление 5 проблем:

1. Кнопка "Именованные области" не работает, удалить раздел из "Свойства ячейки"
2. Проанализировать цвета из 50 макетов и обновить настройки цвета
3. Для новой ячейки кнопки "Параметр/Шаблон" не активны
4. Если шрифт по умолчанию, нет возможности вызвать конструктор шрифта
5. Добавить возможность выделять несколько строк/столбцов

- Проверить, что `onShowNamedAreas` правильно передается в `TemplateToolbar`
- Убедиться, что `showNamedAreasDialog` правильно управляется
- Проверить, что `NamedAreasDialog` корректно рендерится

**Файл**: `src/webview/components/TemplateEditor/TemplatePropertiesPanel.tsx`**Действие**: Удалить раздел "Именованные области" из панели свойств ячейки.**Проверка**: Найти и удалить UI-элементы, связанные с именованными областями в свойствах ячейки (кнопки создания/редактирования/удаления именованных областей).

### 2. Анализ и обновление настроек цвета

**Действие**:

- Проанализировать цвета из макетов в `D:\1C\BUH\src\cf\CommonTemplates`
- Найти все уникальные значения `textColor` и `backColor`
- Обновить `ColorPickerDialog` для поддержки найденных цветов

**Файл**: `src/webview/components/TemplateEditor/ColorPickerDialog.tsx`**Изменения**:

**Примечание**: Нужно проанализировать примеры макетов для понимания всех возможных форматов цветов.

### 3. Исправление активности кнопок "Параметр/Шаблон" для новой ячейки

**Файл**: `src/webview/components/TemplateEditor/TemplatePropertiesPanel.tsx`**Проблема**: Для новой (пустой) ячейки кнопки формата заполнения не активны.**Причина**: Вероятно, проверка `selectedCell` или логика определения типа ячейки не учитывает пустые ячейки.**Исправление**:

- Проверить логику в `FillPatternToggle` или компоненте свойств
- Убедиться, что кнопки активны даже для пустой ячейки
- Возможно, нужно использовать `getCellFillPattern` с обработкой случая пустой ячейки (возвращает 'none', но кнопки должны быть активны)

### 4. Вызов конструктора шрифта для шрифта по умолчанию

**Файл**: `src/webview/components/TemplateEditor/TemplatePropertiesPanel.tsx`**Проблема**: Если у ячейки шрифт по умолчанию (нет индивидуального шрифта), кнопка конструктора шрифта не работает или не отображается.**Исправление**:

- Убедиться, что кнопка "Конструктор шрифта" всегда видна и активна
- При открытии конструктора, если шрифт по умолчанию, создавать новый шрифт для ячейки
- Использовать `updateCellFont` который должен создавать новый шрифт при необходимости

**Проверка**:

- Найти кнопку конструктора шрифта в UI
- Проверить условие `disabled` или `hidden`
- Убедиться, что при отсутствии шрифта создается новый
