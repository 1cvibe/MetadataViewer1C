# Исправление проблем редактора макетов

## Обзор

План включает исправление 4 проблем:

1. Цвета применяются ко всем ячейкам с одинаковым formatIndex - нужно создавать новый формат для каждой ячейки при изменении цвета
2. Нет конструктора шрифтов (FontBuilderDialog уже есть, но нужно убедиться что он работает правильно)
3. Именованные области сделать отдельной кнопкой на панели инструментов и отдельным конструктором (диалог со списком)
4. Для пустой ячейки не работают кнопки Параметр/Шаблон - нужно создавать ячейку если её нет

## Анализ проблемы с цветами

Из примера XML файла видно:

- Несколько ячеек могут использовать один и тот же `formatIndex` (например, `<f>5</f>`)
- Формат определяется в массиве `<format>` по индексу
- Проблема: `updateCellFormat` обновляет существующий формат вместо создания нового

**Решение**: При изменении цвета (или других свойств формата) нужно:

- Создавать новый формат вместо обновления существующего
- Копировать все свойства текущего формата в новый
- Назначать новый formatIndex ячейке
- Это гарантирует, что изменения не затронут другие ячейки

## Изменения

### 1. Исправление применения цветов к ячейкам

**Файл**: `src/utils/templateUtils.ts`**Функция**: `updateCellFormat`**Проблема**: Функция обновляет существующий формат в массиве `template.format[formatIndex]`, что затрагивает все ячейки, использующие этот formatIndex.**Исправление**:

- Всегда создавать новый формат при обновлении (копировать текущий формат и добавить новые свойства)
- Добавить новый формат в массив форматов
- Назначить новый formatIndex ячейке
- Это гарантирует изоляцию изменений для каждой ячейки

**Альтернативный подход** (более сложный, но оптимальный):

- Проверять, используется ли текущий formatIndex другими ячейками
- Если используется другими ячейками - создавать новый формат
- Если используется только этой ячейкой - обновлять существующий

**Рекомендация**: Использовать простой подход - всегда создавать новый формат при изменении, чтобы гарантировать изоляцию.

### 2. Проверка конструктора шрифтов

**Файл**: `src/webview/components/TemplateEditor/FontBuilderDialog.tsx`**Проверка**:

- Убедиться, что компонент правильно работает
- Сравнить структуру данных из примера XML с тем, что сохраняется

**Действие**: Если все работает правильно, можно пропустить этот пункт. Если нет - исправить.

### 4. Исправление работы с пустыми ячейками

**Файл**: `src/utils/templateUtils.ts`**Функции**: `setCellAsParameter`, `setCellAsTemplate`**Проблема**: Функции проверяют `findCellByPosition`, и если ячейка не найдена, возвращают template без изменений.**Исправление**:

- Если ячейка не найдена, создавать её
- Создавать структуру строки, если её нет
- Создавать массив ячеек, если его нет
- Добавлять новую ячейку с правильным индексом колонки

**Логика**:

1. Проверить, существует ли строка `template.rowsItem[row]`
2. Если нет - создать
3. Проверить, существует ли `row.c` (массив ячеек)
4. Если нет - создать пустой массив
5. Добавить новую ячейку с правильным индексом `i` (если нужно)
6. Установить parameter или tl

## Зависимости

- Исправление цветов не зависит от других изменений
- Проверка конструктора шрифтов не зависит от других изменений
- Отдельная кнопка именованных областей не зависит от других изменений
- Исправление пустых ячеек влияет на работу `setCellAsParameter` и `setCellAsTemplate`, которые используются в других местах

## Риски

- **Исправление цветов**: Создание нового формата для каждой ячейки может привести к большому количеству форматов в массиве. Это нормально для 1С, но нужно убедиться, что нет проблем с производительностью.
- **Исправление пустых ячеек**: Нужно правильно определять индекс колонки при создании новой ячейки. Если в строке уже есть ячейки, нужно правильно вычислить индекс.
- **Именованные области**: Новый диалог должен корректно работать с существующими обработчиками и не дублировать функциональность.

## Примечания

- При создании нового формата нужно копировать ВСЕ свойства текущего формата (ширина, высота, выравнивание, границы, шрифт и т.д.), а не только изменяемые свойства
