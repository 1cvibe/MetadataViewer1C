# План оптимизации размера node_modules и webpack bundle

## Текущее состояние

- **Размер node_modules**: ~206 МБ, 229 директорий
- **Размер webpack bundle**: 12.4 МБ (metadataEditor.bundle.js)
- **Проблемы**:
- Минификация отключена (`minimize: false`)
- Нет tree-shaking для некоторых зависимостей
- Неиспользуемые зависимости (`adm-zip`)
- Monaco Editor загружается полностью, хотя используется частично

## Анализ зависимостей

### Используемые в webpack bundle (webview):

- `react`, `react-dom` - используются везде
- `monaco-editor` - используется в XmlEditor, QueryEditor, DcsEditor
- `@rjsf/core`, `@rjsf/utils`, `@rjsf/validator-ajv8` - используются в FormEditor
- `@xmldom/xmldom` - используется в webview компонентах
- `@vscode/webview-ui-toolkit` - используется в webview

### Используемые только в extension host (не в webpack):

- `fast-glob` - используется только в `src/metadataView.ts` (extension host)
- `fast-xml-parser` - используется в extension host
- `adm-zip` - **НЕ ИСПОЛЬЗУЕТСЯ** (можно удалить!)

## План оптимизации

### 1. Удаление неиспользуемых зависимостей

**Файл**: `package.json`

- Удалить `adm-zip` из dependencies (не используется в коде)
- Удалить `@types/adm-zip` из devDependencies

**Ожидаемый эффект**: Уменьшение размера node_modules на несколько МБ

### 2. Оптимизация webpack конфигурации

**Файл**: `webpack.config.js`

#### 2.1. Включить минификацию

- Изменить `minimize: false` на `minimize: true`
- Добавить настройки минификатора для лучшей оптимизации

#### 2.2. Настроить tree-shaking

- Убедиться, что `sideEffects: false` в package.json для зависимостей
- Настроить `usedExports: true` в optimization

#### 2.3. Исключить зависимости extension host из bundle

- Добавить в `externals` зависимости, которые используются только в extension host:
- `fast-glob`
- `fast-xml-parser` (если не используется в webview)

#### 2.4. Оптимизация Monaco Editor

- Использовать только необходимые языки и функции Monaco
- Настроить alias для Monaco, чтобы исключить неиспользуемые части

**Ожидаемый эффект**: Уменьшение bundle на 30-50% (с 12.4 МБ до ~6-8 МБ)

### 3. Оптимизация package.json

**Файл**: `package.json`

- Добавить `"sideEffects": false` для включения tree-shaking
- Проверить версии зависимостей на возможность обновления до более легких версий

### 4. Анализ размера bundle

**Действие**: После оптимизации запустить анализ bundle

- Использовать `webpack-bundle-analyzer` для визуализации размера модулей
- Определить самые большие зависимости
- Принять решение о дальнейшей оптимизации (lazy loading, code splitting для отдельных редакторов)

## Ожидаемые результаты

- **node_modules**: уменьшение на 5-10 МБ (удаление adm-zip и его зависимостей)
- **webpack bundle**: уменьшение на 40-60% (с 12.4 МБ до 5-7 МБ) за счет минификации и tree-shaking
- **Время сборки**: может немного увеличиться из-за минификации, но приемлемо

## Риски

- Минификация может скрыть некоторые ошибки (решается тестированием)
- Tree-shaking может удалить нужный код, если зависимости не помечены правильно
- Изменения в webpack могут повлиять на работу Monaco Editor (нужно протестировать)

## Следующие шаги (опционально)

1. Внедрить lazy loading для отдельных редакторов (FormEditor, DcsEditor, TemplateEditor)
2. Использовать динамические импорты для Monaco Editor
3. Рассмотреть замену @rjsf на более легкую альтернативу, если возможно
4. Оптимизировать React компоненты (memo, useMemo где необходимо)
