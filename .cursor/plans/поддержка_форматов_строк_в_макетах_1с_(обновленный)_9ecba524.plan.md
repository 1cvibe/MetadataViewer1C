# Поддержка форматов строк в макетах 1С (обновленный)

## Проблема

В 1С для разных строк можно задавать свой формат строк через `columnsID`. Например, для строк 14-16 (индекс 13-15) задан свой формат строк с ID `383507ba-2aba-419a-a0fa-73e45b654b25`, а остальные строки используют формат по умолчанию (без `columnsID`). Каждый формат строк имеет:

- Свои именованные области (привязываются через `columnsID` в элементе `area` именованной области)
- Свои ширины колонок
- Свою группу колонок (`TemplateColumns`)

Сейчас в `TemplateTable.tsx` не учитывается `columnsID` при рендеринге, поэтому все строки используют одну группу колонок и одни именованные области.

## Решение

### 1. Обновление интерфейсов для именованных областей

**Файл:** `src/templatInterfaces.ts`Добавить поле `columnsID` в интерфейс `NamedItemArea`:

```typescript
export interface NamedItemArea {
    type: 'Rectangle' | 'Rows' | 'Columns' | 'Row' | 'Column' | string;
    beginRow: number;
    endRow: number;
    beginColumn: number;
    endColumn: number;
    columnsID?: string;  // Привязка к формату строк
}
```



### 2. Определение формата строки для каждой строки

**Файл:** `src/webview/components/TemplateEditor/TemplateTable.tsx`Добавить функцию для определения группы колонок для конкретной строки:

- Если `row.columnsID` есть, найти группу колонок с таким же `id`
- Если `row.columnsID` нет, использовать группу колонок без `id` (формат по умолчанию)
- Если группа не найдена, использовать формат по умолчанию
```typescript
const getColumnsForRow = useCallback((row: TemplateRow): TemplateColumns | null => {
    if (!templateDocument.columns || templateDocument.columns.length === 0) {
        return null;
    }
    
    const rowColumnsID = row.row.columnsID;
    
    if (rowColumnsID) {
        // Ищем группу колонок с таким же ID
        const columnGroup = templateDocument.columns.find(col => col.id === rowColumnsID);
        return columnGroup || getDefaultColumnsGroup();
    } else {
        // Используем группу колонок без ID (формат по умолчанию)
        return getDefaultColumnsGroup();
    }
}, [templateDocument]);

const getDefaultColumnsGroup = useCallback((): TemplateColumns | null => {
    if (!templateDocument.columns || templateDocument.columns.length === 0) {
        return null;
    }
    // Ищем группу колонок без ID (формат по умолчанию)
    const defaultGroup = templateDocument.columns.find(col => !col.id);
    return defaultGroup || templateDocument.columns[0] || null;
}, [templateDocument]);
```




### 3. Состояние для активной строки (hover/selection)

**Файл:** `src/webview/components/TemplateEditor/TemplateTable.tsx`Добавить состояние для отслеживания активной строки, на которую наведен курсор или которая выделена:

```typescript
const [activeRowIndex, setActiveRowIndex] = useState<number | null>(null);
```

Активная строка определяется как:

- Строка, на которую наведен курсор (hover)
- Или строка с выделенной ячейкой (selectedCell.row)

### 4. Получение ширины колонки с учетом формата строки

**Файл:** `src/webview/components/TemplateEditor/TemplateTable.tsx`Модифицировать функцию `getColumnWidth` для принятия группы колонок:

- Принимать `columnsGroup: TemplateColumns | null` как параметр
- Искать колонку в переданной группе
- Если ширина не найдена, возвращать `'9px'` (ширина по умолчанию)
```typescript
const getColumnWidth = React.useCallback((col: number, columnsGroup: TemplateColumns | null): string => {
    if (!columnsGroup || !columnsGroup.columnsItem) {
        return '9px'; // Ширина по умолчанию
    }
    
    const columnItem = columnsGroup.columnsItem.find(item => item.index === col);
    if (columnItem && columnItem.column && columnItem.column.formatIndex !== undefined) {
        const formatIndex = columnItem.column.formatIndex - 1;
        if (templateDocument.format && templateDocument.format[formatIndex]) {
            const format = templateDocument.format[formatIndex];
            if (format.width !== undefined && format.width !== null) {
                if (typeof format.width === 'number') {
                    return `${format.width}px`;
                }
                return format.width;
            }
        }
    }
    
    return '9px'; // Ширина по умолчанию
}, [templateDocument]);
```




### 5. Получение именованных областей с учетом формата строки

**Файл:** `src/utils/templateUtils.ts`Модифицировать функции работы с именованными областями для учета формата строк:

- Добавить параметр `columnsID?: string` в функции `getNamedAreaForRow`, `getNamedAreasForRow`
- Фильтровать именованные области по `columnsID`:
- Если `columnsID` задан, искать области, где `area.columnsID === columnsID`
- Если `columnsID` не задан, искать области, где `area.columnsID` отсутствует или равен `undefined`
```typescript
export function getNamedAreasForRow(
    template: TemplateDocument, 
    row: number,
    columnsID?: string
): NamedArea[] {
    const allAreas = getAllNamedAreas(template);
    const result: NamedArea[] = [];
    
    allAreas.forEach((area, name) => {
        // Проверяем, попадает ли строка в область
        const inRowRange = (area.startRow === -1 || area.startRow <= row) && 
                          (area.endRow === -1 || area.endRow >= row);
        
        if (inRowRange) {
            // Проверяем соответствие columnsID
            const areaColumnsID = getAreaColumnsID(template, name);
            if (columnsID) {
                // Если для строки задан columnsID, ищем области с таким же columnsID
                if (areaColumnsID === columnsID) {
                    result.push(area);
                }
            } else {
                // Если для строки нет columnsID, ищем области без columnsID
                if (!areaColumnsID) {
                    result.push(area);
                }
            }
        }
    });
    
    return result;
}
```




### 6. Динамическое переключение визуального представления

**Файл:** `src/webview/components/TemplateEditor/TemplateTable.tsx`Реализовать динамическое переключение ширины колонок и именованных областей при наведении/выделении:**6.1. Обработчики событий для строк:**

```typescript
const handleRowMouseEnter = useCallback((rowIndex: number) => {
    setActiveRowIndex(rowIndex);
}, []);

const handleRowMouseLeave = useCallback(() => {
    // Не сбрасываем, если строка выделена
    if (selectedCell?.row !== activeRowIndex) {
        setActiveRowIndex(null);
    }
}, [selectedCell, activeRowIndex]);
```

**6.2. Определение активной строки:**

```typescript
const getActiveRowIndex = useCallback((): number | null => {
    // Приоритет: выделенная ячейка > наведение курсора
    if (selectedCell) {
        return selectedCell.row;
    }
    return activeRowIndex;
}, [selectedCell, activeRowIndex]);
```

**6.3. Применение стилей для активной строки:**

- При рендеринге строки проверять, является ли она активной
- Если активна, применять ширины колонок из её формата
- Если не активна, применять ширины из формата по умолчанию
- Аналогично для именованных областей

**6.4. CSS для плавного перехода:**Добавить CSS transition для плавного изменения ширины колонок:

```css
.template-table-cell {
    transition: width 0.2s ease;
}
```

### 8. Обновление парсера XML

**Файл:** `src/metadataView.ts` (или соответствующий парсер)Убедиться, что при парсинге XML макета извлекается `columnsID` из элемента `area` именованной области и сохраняется в `NamedItemArea.columnsID`.

## Риски

1. **Производительность**: Динамическое изменение стилей при наведении может замедлить рендеринг больших таблиц
2. **UX**: Пользователь может не понять, почему ширина колонок меняется при наведении
3. **Совместимость**: Нужно проверить, что все существующие макеты корректно работают с форматом по умолчанию
