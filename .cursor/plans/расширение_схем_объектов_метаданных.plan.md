## Технические детали

### Используемые файлы
- `src/schemas/objectSchemas.ts` - основной файл для расширения
- `src/schemas/baseSchema.ts` - базовые схемы и вспомогательные
- `src/xmlParsers/metadataParser.ts` - парсер XML
- `src/Metadata/metadata-types.ts` - справочник типов объектов
- `scripts/scan-results.json` - результаты сканирования

### Структура новой схемы
Каждая новая схема будет следовать паттерну существующих:
```typescript
export const reportSchema: JSONSchema7 = {
  type: 'object',
  title: 'Отчет',
  properties: {
    ...basePropertiesSchema.properties,
    // Специфичные свойства
    attributes: { ... },
    forms: { ... },
    commands: { ... }
  },
  required: ['Name']
};
````


### Обработка общих элементов

Многие типы объектов имеют общие элементы:

- `attributes` - реквизиты (использовать `attributeSchema`)
- `tabularSections` - табличные части (использовать `tabularSectionSchema`)
- `forms` - формы (использовать `formSchema`)
- `commands` - команды (использовать `commandSchema`)

### Специфичные свойства

Для каждого типа нужно определить уникальные свойства:

- Регистры: `Dimensions`, `Resources`, `Measures`
- Планы счетов: `ChartOfAccountsType`, `AccountCodeLength` и т.д.
- Отчеты/Обработки: специфичные настройки
