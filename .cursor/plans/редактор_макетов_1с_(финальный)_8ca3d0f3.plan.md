#Реализация полноценного редактора макетов 1С (финальный)

## Текущее состояние

- `TemplatePanel` ([src/templatePanel.ts](src/templatePanel.ts)) - только просмотр макетов (read-only)
- `openTemplate` в [src/metadataView.ts](src/metadataView.ts) - парсинг XML и открытие панели
- Структура макетов: `document` с `columns`, `rowsItem`, `format`, `merge`, `font`
- Формат файлов: `Ext/Template.xml` (XML) или `Template.mxlx` (EDT)

## Структура XML макета (на основе реального примера)

### Именованные области

Хранятся как элементы `<namedItem xsi:type="NamedItemCells">` на уровне `document`:

```xml
<namedItem xsi:type="NamedItemCells">
    <name>Данные_Количества</name>
    <area>
        <type>Rectangle</type>
        <beginRow>3</beginRow>
        <endRow>3</endRow>
        <beginColumn>2</beginColumn>
        <endColumn>2</endColumn>
    </area>
</namedItem>
```

**Структура:**

- `namedItem` - элемент с атрибутом `xsi:type="NamedItemCells"`
- `name` - имя области (например, "Данные_Количества")
- `area` - описание области:
- `type` - тип области (Rectangle, Row, Column и т.д.)
- `beginRow` / `endRow` - границы строк (начинается с 1 или 0?)
- `beginColumn` / `endColumn` - границы колонок (начинается с 1 или 0?)

**Расположение:** Элементы `<namedItem>` находятся на том же уровне, что и `<rowsItem>`, `<columns>`, `<format>`, `<merge>`, `<font>`.

### Формат заполнения ячейки

Ячейка может иметь два формата заполнения:**1. Формат заполнения = "параметр" (строки 125-128):**

```xml
<c>
    <f>4</f>
    <parameter>РасчетнаяЦена</parameter>
</c>
```



- Есть элемент `<parameter>` с именем параметра
- Нет элемента `<tl>` (текстовое содержимое) или он пустой
- В ячейке отображается как `<ИмяПараметра>` или `[ИмяПараметра]` в зависимости от настройки

**2. Формат заполнения = "шаблон" (строки 131-139):**

```xml
<c>
    <f>5</f>
    <tl>
        <v8:item>
            <v8:lang>ru</v8:lang>
            <v8:content>расчетная цена - [Цена]</v8:content>
        </v8:item>
    </tl>
</c>
```



- НЕТ элемента `<parameter>`
- Есть элемент `<tl>` с текстовым содержимым
- Текст содержит параметр(ы) в квадратных скобках: `[Цена]`, `расчетная цена - [Цена]`
- В ячейке отображается полный текст с параметрами

**Различие:**

- **"Параметр"** - ячейка содержит только параметр (без статического текста)
- **"Шаблон"** - ячейка содержит текст с встроенными параметрами (шаблон)

## Важные детали из документации и эксперта 1С

### Параметры

- **Синтаксис в макете:** `[ИмяПараметра]` (в квадратных скобках)
- **Формат "параметр":** В XML хранится как `<parameter>ИмяПараметра</parameter>` (элемент, не атрибут!)
- **Формат "шаблон":** В XML хранится как текст в `<tl><v8:item><v8:content>расчетная цена - [Цена]</v8:content></v8:item></tl>`
- **При загрузке:**
- Если есть `<parameter>` → формат "параметр", отображать как `[ИмяПараметра]`
- Если нет `<parameter>`, но есть `<tl>` с `[Имя]` → формат "шаблон", отображать текст как есть
- **При сохранении:**
- Если формат "параметр" (только `[ИмяПараметра]`) → `<parameter>ИмяПараметра</parameter>`, без `<tl>`
- Если формат "шаблон" (текст с `[Имя]`) → `<tl>` с полным текстом, без `<parameter>`

### Именованные области

- **Назначение:** Именованные фрагменты макета для программного управления (циклы, копирование, скрытие)
- **Создание:** Выделить диапазон ячеек → ввести имя в панели свойств
- **Имена должны быть уникальными** в пределах одного макета
- **Использование в BSL:** `Макет.ПолучитьОбласть("ИмяОбласти")` → заполнение параметров → `ТабличныйДокумент.Вывести(Область)`
- **Хранение:** Элементы `<namedItem xsi:type="NamedItemCells">` на уровне `document`

### Объединение ячеек

- **Важно:** При объединении текст берется из **верхней левой ячейки** выделенной области
- Содержимое остальных ячеек **теряется** при объединении
- При программном доступе обращаться к верхней левой ячейке объединенной области

### 3. Утилиты для работы с макетами

**Файл**: [src/utils/templateUtils.ts](src/utils/templateUtils.ts) (новый)**Основные функции:**

- `updateTemplateDocument(original: TemplateDocument, changes: Partial<TemplateDocument>): TemplateDocument`

- `addColumn(template: TemplateDocument, index: number, position: 'left' | 'right'): TemplateDocument`
- `deleteColumn(template: TemplateDocument, index: number): TemplateDocument`
- `mergeCells(template: TemplateDocument, startRow, startCol, endRow, endCol): TemplateDocument`
- **Важно:** При объединении текст берется из верхней левой ячейки (startRow, startCol)
- Содержимое остальных ячеек теряется
- При объединении нужно учитывать формат заполнения верхней левой ячейки
- `unmergeCells(template: TemplateDocument, row, col): TemplateDocument`
- `insertCell(template: TemplateDocument, row, col, direction: 'horizontal' | 'vertical'): TemplateDocument`
- `deleteCell(template: TemplateDocument, row, col, direction: 'horizontal' | 'vertical'): TemplateDocument`
- `updateCellFormat(template: TemplateDocument, row, col, format: Partial<TemplateFormat>): TemplateDocument`
- `updateCellFont(template: TemplateDocument, row, col, font: Partial<TemplateFont>): TemplateDocument`
- `updateCellText(template: TemplateDocument, row, col, text: string): TemplateDocument`

**Функции для работы с форматами заполнения:**

- `setCellAsParameter(template: TemplateDocument, row, col, parameterName: string): TemplateDocument`
- Устанавливает формат "параметр": создает `<parameter>Имя</parameter>`, удаляет `<tl>`
- `setCellAsTemplate(template: TemplateDocument, row, col, templateText: string): TemplateDocument`
- Устанавливает формат "шаблон": создает `<tl>` с текстом, удаляет `<parameter>`
- `getCellFillPattern(template: TemplateDocument, row, col): 'parameter' | 'template' | 'none'`
- Определяет формат заполнения ячейки:
    - Если есть `<parameter>` → 'parameter'
    - Если нет `<parameter>`, но есть `<tl>` с текстом → 'template'
    - Иначе → 'none'
- `toggleCellFillPattern(template: TemplateDocument, row, col): TemplateDocument`
- Переключает формат заполнения:
    - Если 'parameter' → преобразует в 'template' (извлекает имя параметра и создает текст `[Имя]`)
    - Если 'template' → пытается извлечь единственный параметр из текста и преобразует в 'parameter'
- `extractParameterFromTemplate(templateText: string): string | null`
- Извлекает имя параметра из шаблона вида `[ИмяПараметра]` или `текст [ИмяПараметра]`
- Возвращает null, если параметров несколько или их нет
- `parseTemplateParameters(templateText: string): string[]`
- Извлекает все параметры из шаблона (все вхождения `[Имя]`)
- `convertParameterToTemplate(parameterName: string): string`
- Преобразует имя параметра в шаблон: `ИмяПараметра` → `[ИмяПараметра]`

**Функции для именованных областей:**

- `createNamedArea(template: TemplateDocument, name: string, areaType: 'Rectangle' | 'Row' | 'Column', startRow, startCol, endRow, endCol): TemplateDocument`
- Создает `<namedItem xsi:type="NamedItemCells">` на уровне `document`
- Валидирует уникальность имени
- `deleteNamedArea(template: TemplateDocument, name: string): TemplateDocument`
- Удаляет соответствующий `<namedItem>` элемент
- `renameNamedArea(template: TemplateDocument, oldName: string, newName: string): TemplateDocument`
- Обновляет `<name>` в соответствующем `<namedItem>`
- `updateNamedArea(template: TemplateDocument, name: string, updates: Partial<NamedArea>): TemplateDocument`
- Обновляет область, координаты или тип
- `getNamedArea(template: TemplateDocument, name: string): NamedArea | null`
- `getAllNamedAreas(template: TemplateDocument): Map<string, NamedArea>`
- `validateNamedAreaName(template: TemplateDocument, name: string): {valid: boolean, error?: string}`
- `findNamedAreaByPosition(template: TemplateDocument, row, col): NamedArea[]`
- Находит все именованные области, которые содержат указанную ячейку

**Вспомогательные функции:**

- `findCellByPosition(template: TemplateDocument, row, col): TemplateCell | null`
- `getEffectiveFormat(template: TemplateDocument, row, col): TemplateFormat`
- `getEffectiveFont(template: TemplateDocument, row, col): TemplateFont | null`
- `updateMergeAfterStructureChange(template: TemplateDocument): TemplateDocument`
- `validateTemplate(template: TemplateDocument): {valid: boolean, errors: string[]}`
- `normalizeTemplateIndexes(template: TemplateDocument): TemplateDocument` - нормализация индексов после изменений

### 4. Сохранение в XML

**Файл**: [src/utils/templateXmlSaver.ts](src/utils/templateXmlSaver.ts) (новый)

- `saveTemplateToXml(templateDocument: TemplateDocument, originalXml: string, templatePath: string): string`
- Использование `@xmldom/xmldom` для сохранения структуры (аналогично `applyFormChangesToXmlStringWithDom`)
- Сохранение BOM (как в `dcsEditor.ts:612`)

**Корректное сохранение:**

1. **Форматов (`format`)** с обновлением индексов
2. **Шрифтов (`font`)** с обновлением индексов
3. **Объединенных ячеек (`merge`)**
4. **Колонок (`columns`)**
5. **Строк (`rowsItem`)**
6. **Параметров ячеек:**

- **Формат "параметр":** Сохранять как `<parameter>ИмяПараметра</parameter>` (элемент, не атрибут!)
- **Формат "шаблон":** Сохранять как `<tl><v8:item><v8:lang>ru</v8:lang><v8:content>текст [Имя]</v8:content></v8:item></tl>`
- При переключении формата: удалять противоположный элемент

7. **Именованных областей (`namedItem`):**

- Сохранять как элементы `<namedItem xsi:type="NamedItemCells">` на уровне `document`
- Структура:
     ```xml
               <namedItem xsi:type="NamedItemCells">
                   <name>ИмяОбласти</name>
                   <area>
                       <type>Rectangle</type>
                       <beginRow>...</beginRow>
                       <endRow>...</endRow>
                       <beginColumn>...</beginColumn>
                       <endColumn>...</endColumn>
                   </area>
               </namedItem>
     ```


**Обработка параметров при сохранении:**

- Если формат "параметр": `cellData.parameter = "Имя"` → `<parameter>Имя</parameter>`, без `<tl>`
- Если формат "шаблон": `cellData.tl = { text: "расчетная цена - [Цена]" }` → `<tl><v8:item><v8:content>расчетная цена - [Цена]</v8:content></v8:item></tl>`, без `<parameter>`

**Обработка параметров при загрузке:**

- Если есть `<parameter>ИмяПараметра</parameter>` → формат "параметр", отображать `[ИмяПараметра]`
- Если нет `<parameter>`, но есть `<tl><v8:item><v8:content>текст [Имя]</v8:content></v8:item></tl>` → формат "шаблон", отображать текст как есть

```typescript
    type: 'Rectangle' | 'Row' | 'Column' | string;  // Тип области
    beginRow: number;     // Начальная строка (проверить, с 0 или с 1)
    endRow: number;       // Конечная строка
    beginColumn: number;  // Начальная колонка (проверить, с 0 или с 1)
    endColumn: number;    // Конечная колонка
}

interface TemplateCellData {
    f: number;              // Индекс формата
    parameter?: string;     // Имя параметра (формат "параметр") - элемент <parameter>
    tl?: TemplateTextData;  // Текстовое содержимое (формат "шаблон") - элемент <tl>
}

// Уточнение: parameter - это элемент, а не атрибут!
// В XML: <parameter>ИмяПараметра</parameter>
// В парсере: parameter: "ИмяПараметра" (текстовое содержимое элемента)

// Вспомогательные типы
interface CellPosition { row: number; col: number; }
interface CellRange { startRow: number; startCol: number; endRow: number; endCol: number; }
interface NamedArea { name: string; areaType: string; startRow: number; startCol: number; endRow: number; endCol: number; }
interface FormatBuilder { type: 'number' | 'date' | 'boolean'; options?: any; }
```

**Важно о параметрах:**

- В XML параметр хранится как элемент `<parameter>ИмяПараметра</parameter>` (не атрибут!)
- При парсинге: `parameter: "ИмяПараметра"` (текстовое содержимое элемента)
- При редактировании в UI отображается как `[ИмяПараметра]`
- При сохранении: `parameter = "ИмяПараметра"` → `<parameter>ИмяПараметра</parameter>`
- При загрузке: `<parameter>ИмяПараметра</parameter>` → `parameter = "ИмяПараметра"`, отображение `[ИмяПараметра]`

**Важно о формате заполнения:**

- **"Параметр"**: есть `<parameter>`, нет `<tl>` (или `<tl>` пустой)
- **"Шаблон"**: нет `<parameter>`, есть `<tl>` с текстом, содержащим `[ИмяПараметра]`
- При переключении формата нужно преобразовывать содержимое

## Риски и ограничения

- Сложная структура XML макетов требует точного сохранения порядка и атрибутов
- Параметры - это элементы `<parameter>`, а не атрибуты (важно для парсинга и сохранения!)
- Формат заполнения определяется наличием элементов `<parameter>` и `<tl>`:
- "Параметр": есть `<parameter>`, нет `<tl>`
- "Шаблон": нет `<parameter>`, есть `<tl>` с текстом
- Объединенные ячейки - сложность при редактировании (текст из верхней левой ячейки, формат заполнения сохраняется)
- Именованные области хранятся как `<namedItem xsi:type="NamedItemCells">` на уровне `document`
- Координаты `beginRow/endRow/beginColumn/endColumn` нужно проверить, с 0 или с 1 начинаются
- Тип области `type` может быть "Rectangle", "Row", "Column" и другие
- Форматы/шрифты по индексам - требуется обновление ссылок при удалении
- Различные типы макетов (с `columnsID` и без) требуют различной обработки
