# Улучшения редактора макетов: конструкторы цвета/шрифта, исправление выделения и переключений

## Обзор

План включает:

1. Добавление конструктора цвета (диалог с HEX полем и палитрой)
2. Добавление конструктора шрифта (семейство, размер, стили)
3. Исправление переключения "Параметр"/"Шаблон" (сохранение текущих значений)
4. Удаление кнопок "Вставить параметр"/"Вставить шаблон" из Toolbar
5. Добавление выделения всей строки/колонки по клику на заголовки
6. Исправление логики выделения ячеек

## Изменения

### 1. Конструктор цвета (`ColorPickerDialog.tsx`)

**Файл**: `src/webview/components/TemplateEditor/ColorPickerDialog.tsx` (новый)Создать компонент диалога выбора цвета:

- Поле ввода HEX кода (например, `#000000`)
- Палитра основных цветов (черный, белый, серый, красный, зеленый, синий, желтый и т.д.)
- Поддержка стилей 1С (например, `style:NegativeTextColor`) - поле для ввода стиля
- Кнопки "ОК" и "Отмена"

**Интеграция**:

- В `TemplatePropertiesPanel.tsx` заменить текстовые поля для `textColor` и `backColor` на кнопки, открывающие диалог
- Добавить состояние для открытия/закрытия диалога
- Передать обработчики через пропсы `onColorsChange`

### 2. Конструктор шрифта (`FontBuilderDialog.tsx`)

**Файл**: `src/webview/components/TemplateEditor/FontBuilderDialog.tsx` (новый)Создать компонент диалога выбора шрифта:

- Выбор семейства шрифта (select с популярными шрифтами: Arial, Times New Roman, Courier New и т.д.)
- Поле ввода размера шрифта (number input)
- Чекбоксы для стилей: жирный, курсив, подчеркнутый, зачеркнутый
- Кнопки "ОК" и "Отмена"

**Интеграция**:

- В `TemplatePropertiesPanel.tsx` добавить кнопку "Конструктор шрифта" в секцию "Шрифт"
- Добавить состояние для открытия/закрытия диалога
- При сохранении вызывать `updateCellFont` через `onFontChange`

### 3. Исправление переключения "Параметр"/"Шаблон"

**Файл**: `src/webview/components/TemplateEditor/TemplateEditorApp.tsx`**Проблема**: `handleFillPatternToggle` использует `prompt()` и не учитывает текущие значения из панели свойств.**Исправление**:

- Изменить логику `handleFillPatternToggle`: при переключении на "Параметр" использовать текущий `templateText` (если есть) или `parameterName` из панели
- При переключении на "Шаблон" использовать текущий `parameterName` (обернутый в `[...]`) или `templateText` из панели
- Если значения отсутствуют, создавать пустые (не использовать `prompt`)

**Файл**: `src/webview/components/TemplateEditor/FillPatternToggle.tsx`

- Убедиться, что `onToggle` вызывается только при клике на кнопки, а не при изменении полей ввода

### 4. Удаление кнопок "Вставить параметр"/"Вставить шаблон"

**Файл**: `src/webview/components/TemplateEditor/TemplateToolbar.tsx`

- Удалить пропсы `onInsertParameter` и `onInsertTemplate` из интерфейса `TemplateToolbarProps`
- Удалить соответствующие кнопки из JSX (строки ~150-163)

**Файл**: `src/webview/components/TemplateEditor/TemplateEditorApp.tsx`

- Убедиться, что пропсы `onInsertParameter` и `onInsertTemplate` не передаются в `TemplateToolbar`

### 5. Выделение всей строки/колонки

**Файл**: `src/webview/components/TemplateEditor/TemplateTable.tsx`**Добавить обработчики**:

- `handleRowHeaderClick(row: number)` - выделяет всю строку (создает `CellRange` от колонки 0 до `maxColumns - 1`)
- `handleColumnHeaderClick(col: number)` - выделяет всю колонку (создает `CellRange` от строки 0 до `rows.length - 1`)

**Изменить JSX**:

- Добавить `onClick` обработчики к `<td className="template-table-row-header">` (номер строки)
- Добавить `onClick` обработчики к `<th className="template-table-column-header">` (буква колонки)
- Добавить CSS классы для визуального выделения заголовков при hover

**Файл**: `src/webview/components/TemplateEditor/template-editor.css`

- Добавить стили для `.template-table-row-header:hover` и `.template-table-column-header:hover`
- Добавить стили для выделенных заголовков (например, `.template-table-row-header.selected`)

### 6. Исправление выделения ячеек

**Файл**: `src/webview/components/TemplateEditor/TemplateTable.tsx`**Проблемы** (потенциальные):

- Логика перетаскивания использует фиксированные размеры ячеек (`cellWidth = 100`, `cellHeight = 25`)
- Не учитываются реальные размеры ячеек из `format.width` и `format.height`

**Исправления**:

- Улучшить логику `handleMouseMove` в `useEffect` для перетаскивания: вычислять реальные координаты ячеек на основе таблицы DOM
- Использовать `getBoundingClientRect()` для каждой ячейки или использовать более точный алгоритм определения ячейки по координатам
- Убедиться, что выделение корректно работает с объединенными ячейками

**Дополнительно**:

- Проверить, что `isCellSelected` корректно обрабатывает случаи, когда `selectedCell` и `selectedRange` установлены одновременно
- Убедиться, что при клике на ячейку `selectedRange` сбрасывается в `null`, если не используется Shift+клик

## Зависимости

- Конструктор цвета не зависит от других изменений
- Конструктор шрифта не зависит от других изменений
- Исправление переключения "Параметр"/"Шаблон" должно быть выполнено перед тестированием FillPatternToggle
- Удаление кнопок можно выполнить независимо
- Выделение строк/колонок не зависит от других изменений
- Исправление выделения ячеек может потребовать тестирования с реальными данными

## Риски

- Конструктор цвета: необходимо правильно валидировать HEX коды и стили 1С
- Конструктор шрифта: нужно убедиться, что все свойства шрифта корректно сохраняются
- Выделение строк/колонок: может конфликтовать с существующей логикой выделения (нужно протестировать)
- Исправление выделения ячеек: изменение логики перетаскивания может повлиять на существующее поведение

## Примечания

- Все новые диалоги должны использовать стили VS Code (переменные CSS типа `var(--vscode-dialog-background)`)
