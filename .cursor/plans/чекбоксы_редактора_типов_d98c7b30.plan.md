# План: исправление позиции чекбоксов в редакторе типов

## 1. Причина

**Симптом:** В модалке «Редактор типа» чекбоксы у элементов (в т.ч. «СправочникОбъект.Контрагенты», «СправочникОбъект.Подразделения») отображаются **справа и слегка ниже** текста, а должны быть **слева**, непосредственно перед подписью.

**Корневая причина:** Элементы списка типов — это `<label className="type-item">` с `<input type="checkbox">` и `<span>{label}</span>`. Для них в [editor.css](src/webview/styles/editor.css) (≈387–453) задана flex-раскладка:

- `.type-item { display: flex; flex-direction: row; align-items: center; gap: 8px; }`
- `.type-item input[type="checkbox"] { flex-shrink: 0; ... }`

TypeWidget используется в двух контекстах:

1. **В модалках** (PredefinedTypeEditorModal, CharacteristicTypeEditorModal, AttributeTypeEditorModal и др.): `Modal` → `form-field` → TypeWidget. Модалка — **сосед** формы, не внутри `.rjsf`.
2. **Внутри RJSF Form** (вкладка properties): TypeWidget как `ui:widget` для полей типа. Модалка редактора типов рендерится **внутри** TypeWidget, т.е. внутри `.rjsf`.

В [editor.css](src/webview/styles/editor.css) (1320–1326) задано:

```css
.rjsf label {
  display: block;
  margin-bottom: 8px;
  ...
}
```

Специфичность `.rjsf label` (1 класс + 1 элемент) **выше**, чем у `.type-item` (1 класс). В результате при открытии редактора из контекста RJSF (и при любом совпадении DOM под `.rjsf`) ко всем `label`, включая `label.type-item`, применяется `display: block`. Flex-раскладка перестаёт работать, чекбокс и текст ведут себя как обычный поток — появляется смещение вправо/вниз.

Аналогично могут страдать заголовки категорий («Примитивные типы», «Специальные типы», «Ссылочные типы»): они тоже `label` внутри `.type-category-header`.

**Уже сделано:** В editor.css используется `.form-field > label` (1066–1071), а не `.form-field label`, поэтому прямые подписи полей в модалках не затрагивают вложенные `label` в TypeWidget. Проблему создаёт именно `.rjsf label`.

## 2. Варианты исправления

**Вариант A (рекомендуется): повысить специфичность селекторов редактора типов**

- Добавить контекстные правила для списка типов и заголовков категорий так, чтобы они **гарантированно** перебивали `.rjsf label`:
  - `.type-editor-modal .type-item` — элементы списка (примитивные, специальные, ссылочные, в т.ч. внутри `.type-group-items`);
  - `.type-editor-modal .type-category-header label` — заголовки «Примитивные типы» и т.д.
- В этих правилах явно задать `display: flex`, `flex-direction: row`, `align-items: center`, `gap: 8px` и сохранить стили чекбокса (`flex-shrink: 0`, при необходимости `order: -1`).
- Файл: [editor.css](src/webview/styles/editor.css). Разместить новые правила **после** блока `.rjsf` (после ≈1355), чтобы они имели приоритет при равной специфичности.

**Вариант B:** Сузить область действия `.rjsf label` (например, только для `.rjsf > .form-group > label` или подобного). Это может затронуть другие формы; требуется ревью всех использований `label` в RJSF.

Ориентир: **вариант A** — минимальные, изолированные изменения только для редактора типов.

## 3. Конкретные правки в editor.css

1. **Добавить блок переопределений для редактора типов** (после секции `.rjsf`, ≈1355):

   - `.type-editor-modal .type-item` — восстановить flex, чекбокс слева.
   - `.type-editor-modal .type-category-header label` — оставить flex для заголовков категорий.

2. **Чекбокс в .type-item:**

   - Сохранить `flex-shrink: 0`, при желании добавить `order: -1` для явного размещения слева.

3. **Опционально:** Для единообразия с `.type-list` задать для `.type-group-items` в CSS (вместо только inline-стилей) `display: flex; flex-direction: column; gap: 4px;` и проверить, что чекбоксы в группах «СправочникОбъект» и т.п. выровнены так же, как в примитивных/специальных типах.

4. **Не трогать:** Логику [TypeWidget.tsx](src/webview/widgets/TypeWidget.tsx), разметку `label`/`input`/`span`, вызовы TypeWidget в модалках — только CSS.

- Изменения только в CSS; регрессии маловероятны при аккуратной работе с селекторами.
- Если в проекте появятся другие `label` внутри `.type-editor-modal`, они тоже получат flex. Сейчас там только тип-лист и заголовки категорий — это ожидаемо.
