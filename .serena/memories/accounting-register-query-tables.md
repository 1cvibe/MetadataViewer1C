# Таблицы запросов регистра бухгалтерии

## Виртуальные таблицы для запросов

Регистр бухгалтерии предоставляет следующие виртуальные таблицы:

1. **Движения** - содержит все бухгалтерские проводки (детальные записи)
2. **Остатки** - возвращает состояние счетов на начало или конец указанного периода
3. **Обороты** / **ОборотыД** - возвращает сумму движений по счетам за указанный период (являются синонимами в актуальных версиях платформы)
4. **ОстаткиИОбороты** - универсальная таблица, которая позволяет получить остатки на начало, обороты за период и остатки на конец одним запросом
5. **ДвиженияССубконто** - специфическая таблица, которая "развертывает" проводки с несколькими видами субконто в несколько строк (для детального анализа)
6. **Субконто** и **КорСубконто** - не самостоятельные таблицы, а поля в составе других виртуальных таблиц, содержащие информацию о видах субконто

## Виртуальная таблица Движения

### Назначение
Получение детальной информации по всем бухгалтерским проводкам за указанный период.

### Синтаксис
```
РегистрБухгалтерии.<ИмяРегистра>.Движения(
    &НачалоПериода,
    &КонецПериода,
    СостояниеПериода = <Значение>,
    Условие = <Строка>
) КАК <Псевдоним>
```

### Параметры
- **Начало/Конец периода** - границы временного интервала для выборки проводок (аналогично регистрам накопления)
- **СостояниеПериода** - определяет, какие движения включать:
  - `ЕстьДвижения` - в периоде есть движения
  - `ЕстьДвиженияГраницыПериода` - на границе периода есть движения
- **Условие** - строковое выражение для фильтрации проводок

### Доступные поля
- `Регистратор` - документ, создавший проводку
- `Период` - момент времени проводки
- `НомерСтроки` - номер строки движения
- `Счет` (для регистров без корреспонденции) или `СчетДт`, `СчетКт` (для регистров с корреспонденцией)
- `Субконто1`, `Субконто2`, `Субконто3` - значения субконто
- Все ресурсы регистра (Сумма, Количество и т.д.)
- Все измерения и реквизиты регистра

### Пример
```sql
ВЫБРАТЬ
    ХозрасчетныйДвижения.Регистратор,
    ХозрасчетныйДвижения.Период,
    ХозрасчетныйДвижения.Счет,
    ХозрасчетныйДвижения.Субконто1,
    ХозрасчетныйДвижения.Сумма
ИЗ
    РегистрБухгалтерии.Хозрасчетный.Движения(
        &НачалоПериода,
        &КонецПериода,
        СостояниеПериода = ЕстьДвижения
    ) КАК ХозрасчетныйДвижения
ГДЕ
    ХозрасчетныйДвижения.Счет = &Счет
```

## Виртуальная таблица ДвиженияССубконто

### Назначение
Специфическая таблица для регистров бухгалтерии, которая "развертывает" проводки, у которых несколько видов субконто, в несколько строк - по одной строке для каждого вида субконто. Это полезно для детального анализа проводок по каждому виду субконто в отдельности.

### Синтаксис
```
РегистрБухгалтерии.<ИмяРегистра>.ДвиженияССубконто(
    &НачалоПериода,
    &КонецПериода,
    Условие = <Строка>
) КАК <Псевдоним>
```

### Параметры
- **Начало/Конец периода** - границы временного интервала (аналогично `Движения`)
- **Условие** - строковое выражение для фильтрации

### Доступные поля
Аналогичны таблице `Движения`, но с разверткой по видам субконто:
- Для каждой проводки создается несколько строк - по одной для каждого вида субконто
- Поля для идентификации вида субконто и его значения
- Все остальные поля из таблицы `Движения`

### Пример использования
Используется для построения отчетов, где требуется анализ проводок по каждому виду субконто отдельно.
, которая позволяет получить остатки на начало, обороты за период и остатки на конец одним запросом

## Синтаксис виртуальной таблицы ОстаткиИОбороты

```
РегистрБухгалтерии.<ИмяРегистра>.ОстаткиИОбороты(
    &НачалоПериода,
    &КонецПериода,
    &Периодичность,
    <Измерения>,
    <Ресурсы>,
    <СпособЗаполнения>
) КАК <Псевдоним>
```

### Параметры:
- **Начало/Конец периода** - границы интервала времени (обычно передаются как параметры запроса)
- **Периодичность** - детализация расчета:
  - `РЕГИСТРАЦИЯ` - по каждой записи (построчно)
  - `ДЕНЬ` - итоги за каждый день
  - `МЕСЯЦ` - итоги за каждый месяц
  - `КВАРТАЛ` - итоги за каждый квартал
  - `ГОД` - итоги за каждый год
  - `Авто` - автоматический выбор
- **Измерения** - поля для группировки (например, `Счет`, `Субконто`)
- **Ресурсы** - поля для суммирования (например, `Сумма`, `Количество`)
- **СпособЗаполнения**:
  - `"Движения"` - только поля оборотов (`СуммаОборот`, `КоличествоОборот`)
  - `"Остатки"` - только поля остатков (`СуммаНачальныйОстаток`, `СуммаКонечныйОстаток`)
  - `"ОстаткиИОбороты"` (по умолчанию) - доступны и остатки, и обороты

## Доступные поля в виртуальных таблицах

### Для остатков:
- `СуммаНачальныйОстаток`, `КоличествоНачальныйОстаток`
- `СуммаКонечныйОстаток`, `КоличествоКонечныйОстаток`
- `СуммаНачальныйОстатокДт`, `СуммаНачальныйОстатокКт`
- `СуммаКонечныйОстатокДт`, `СуммаКонечныйОстатокКт`

### Для оборотов:
- `СуммаОборотДт`, `КоличествоОборотДт`
- `СуммаОборотКт`, `КоличествоОборотКт`
- `СуммаОборот`, `КоличествоОборот` (для ОстаткиИОбороты)

### Измерения:
- `Счет` - счет (для регистров без корреспонденции)
- `СчетДт`, `СчетКт` - счета дебета и кредита (для регистров с корреспонденцией)
- `Субконто1`, `Субконто2`, `Субконто3` - значения субконто
- `СубконтоДт1..3`, `СубконтоКт1..3` - значения субконто для корреспонденции
- `Валюта` - если учет ведется в валюте
- Другие измерения, определенные в конфигураторе

## Физические таблицы базы данных

Для каждого регистра бухгалтерии в базе данных создаются следующие физические таблицы:

- **`_AccRg<n>`** - таблица движений регистра бухгалтерии
- **`_AccRgED<n>`** - таблица значений субконто регистра бухгалтерии (создается, если регистр ссылается на план счетов с максимальным количеством субконто > 0)
- **`_AccRgAT0<n>`** - таблица итогов по счету
- **`_AccRgAT<i><n>`** - где i от 1 до максимального количества субконто. Таблица итогов по счету с количеством видов субконто равным i
- **`_AccRgCT<n>`** - таблица итогов оборотов между счетами (только для регистра бухгалтерии, поддерживающего корреспонденцию)
- **`_AccRgChngR<n>`** - таблица регистрации изменений регистра бухгалтерии (создается, если регистр участвует хотя бы в одном плане обмена)
- **`_AccRgOpt`** - таблица настроек хранения итогов (одна на все регистры бухгалтерии)

## Структура таблицы итогов по счету (_AccRgAT<n>)

Содержит поля:
- `_Period` - момент, на который подсчитаны итоги
- `_Account<suff>` - ссылка на счет
- `_Value<i><suff>` - значение субконто (для i от 1 до k)
- `_Fld<n><suff>` - измерение или ресурс (для каждого измерения и каждого ресурса)
- `_TurnoverDt<n>` - дебетовые обороты ресурса (для каждого ресурса)
- `_TurnoverCt<n>` - кредитовые обороты ресурса (для каждого ресурса)
- `_Turnover<n>` - общие обороты ресурса (для каждого ресурса)
- `_Splitter` - разделитель итогов (для обеспечения параллельности проведения документов)
- `_Fld<n>` - общие реквизиты

## Структура таблицы итогов оборотов между счетами (_AccRgCT<n>)

Для регистра бухгалтерии, поддерживающего корреспонденцию:
- `_Period` - момент, на который подсчитаны итоги
- `_AccountDt<suff>` - дебетовый счет
- `_AccountCt<suff>` - кредитовый счет
- `_Fld<n><suff>` - измерение или ресурс (для каждого балансового измерения и каждого балансового ресурса)
- `_Fld<n>Dt<suff>` - дебетовая составляющая измерения или ресурса (для каждого небалансового измерения и каждого небалансового ресурса)
- `_Fld<n>Ct<suff>` - кредитовая составляющая измерения или ресурса (для каждого небалансового измерения и каждого небалансового ресурса)
- `_Splitter` - разделитель итогов
- `_Fld<n>` - общие реквизиты

## Структура таблицы настроек хранения итогов (_AccRgOpt)

Создается одна таблица на все регистры бухгалтерии с полями:
- `_RegID` - идентификатор регистра бухгалтерии
- `_Period` - периодичность хранения итогов (не используется)
- `_ActualPeriod` - хранение актуальных итогов
- `_Periodicity` - периодичность регистра
- `_RepetitionFactor` - кратность (не используется)
- `_UseTotals` - признак использования итогов
- `_MinPeriod` - минимальный период, с которого нужно пересчитывать итоги
- `_UseSplitter` - признак использования разделителя итогов
- `_MinCalculatedPeriod` - минимальный период, по которому нужно насчитывать итоги
- `_Fld<n>` - общие реквизиты

## Регистры бухгалтерии с поддержкой корреспонденции

### Основное отличие регистров с корреспонденцией

Регистры бухгалтерии могут быть настроены на поддержку корреспонденции счетов (двойной записи). Для таких регистров характерны следующие особенности:

**В структуре регистра:**
- Каждая запись содержит обязательные реквизиты **Счет Дт** (счет дебета) и **Счет Кт** (счет кредита)
- Вместо одного поля `Счет` используются два поля: `СчетДт` и `СчетКт`
- Аналогично для субконто: используются `СубконтоДт1..3` и `СубконтоКт1..3`

### Физические таблицы для регистров с корреспонденцией

#### Таблица движений (_AccRg<n>)

Для регистра с корреспонденцией таблица движений содержит:
- **`_AccountDt<suff>`** - ссылка на счет дебета (вместо `_Account`)
- **`_AccountCt<suff>`** - ссылка на счет кредита (вместо `_Account`)
- **`_Correspond`** - поле отсутствует (присутствует только в регистрах без корреспонденции)
- **`_Fld<n><suff>`** - измерение, ресурс или реквизит
- **`_Active`** - признак того, что движение участвует в подсчете итогов
- **`_Period`** - момент времени движения
- **`_Recorder<suff>`** - ссылка на регистратор
- **`_LineNo`** - номер движения по данному регистратору

**Индексы таблицы движений:**
- Индекс по `СчетДт + Период + Регистратор`
- Индекс по `СчетКт + Период + Регистратор`
- (Вместо одного индекса по `Счет` для регистров без корреспонденции)

#### Таблица итогов оборотов между счетами (_AccRgCT<n>)

**Создается ТОЛЬКО для регистров, поддерживающих корреспонденцию.**

Содержит поля:
- **`_Period`** - момент, на который подсчитаны итоги
- **`_AccountDt<suff>`** - дебетовый счет
- **`_AccountCt<suff>`** - кредитовый счет
- **`_Fld<n><suff>`** - измерение или ресурс (для каждого **балансового** измерения и каждого **балансового** ресурса)
- **`_Fld<n>Dt<suff>`** - дебетовая составляющая измерения или ресурса (для каждого **небалансового** измерения и каждого **небалансового** ресурса)
- **`_Fld<n>Ct<suff>`** - кредитовая составляющая измерения или ресурса (для каждого **небалансового** измерения и каждого **небалансового** ресурса)
- **`_Splitter`** - разделитель итогов (для обеспечения параллельности проведения документов)
- **`_Fld<n>`** - общие реквизиты

**Индексы таблицы итогов между счетами:**
- Индекс по балансовым измерениям: `ИзмерениеДт1 + ИзмерениеКт1 + ... Измерение1 + ... [+ ХэшИзмерений] [+ РазделительИтогов]`
- Индекс по небалансовым измерениям: аналогичный, но с разделением на Дт и Кт составляющие

### Особенности использования виртуальных таблиц для регистров с корреспонденцией

#### 1. Автоматическая обработка корреспонденции

Виртуальные таблицы автоматически обрабатывают обе стороны проводки. При использовании условия:
```sql
Счет В ИЕРАРХИИ (&Счет)
```
Платформа автоматически преобразует это в:
```sql
(СчетДт = &Счет ИЛИ СчетКт = &Счет)
```

#### 2. Поля корреспонденции в виртуальных таблицах

Для регистров с корреспонденцией доступны специальные поля:
- **`КорСчет`** - корреспондирующий счет (счет с противоположной стороны проводки)
- **`КорСубконто1..3`** - корреспондирующие субконто
- Если основной счет был в Дебете, то `КорСчет` будет `СчетКт`, и наоборот

#### 3. Поля остатков и оборотов с разделением по Дт/Кт

- **`СуммаНачальныйОстатокДт`**, **`СуммаНачальныйОстатокКт`** - остатки на начало периода
- **`СуммаКонечныйОстатокДт`**, **`СуммаКонечныйОстатокКт`** - остатки на конец периода
- **`СуммаОборотДт`**, **`СуммаОборотКт`** - обороты по дебету и кредиту
- **`КоличествоОборотДт`**, **`КоличествоОборотКт`** - обороты по количеству

### Примеры запросов для регистров с корреспонденцией

#### Пример 1: Остатки с использованием иерархии счетов
```sql
ВЫБРАТЬ
    ХозрасчетныйОстатки.Счет,
    ХозрасчетныйОстатки.Субконто1,
    ХозрасчетныйОстатки.СуммаНачальныйОстатокДт,
    ХозрасчетныйОстатки.СуммаНачальныйОстатокКт,
    ХозрасчетныйОстатки.СуммаКонечныйОстатокДт,
    ХозрасчетныйОстатки.СуммаКонечныйОстатокКт
ИЗ
    РегистрБухгалтерии.Хозрасчет.Остатки(
        &Период,
        Счет В ИЕРАРХИИ (&Счет)  // Автоматически обрабатывает СчетДт и СчетКт
    ) КАК ХозрасчетныйОстатки
```

#### Пример 2: Обороты с анализом корреспонденции
```sql
ВЫБРАТЬ
    ХозрасчетныйОбороты.Счет,
    ХозрасчетныйОбороты.Субконто1 КАК Номенклатура,
    ХозрасчетныйОбороты.КорСчет КАК КорреспондирующийСчет,
    ХозрасчетныйОбороты.КорСубконто1 КАК Контрагент,
    ХозрасчетныйОбороты.СуммаОборотДт,
    ХозрасчетныйОбороты.СуммаОборотКт
ИЗ
    РегистрБухгалтерии.Хозрасчет.Обороты(
        &НачалоПериода,
        &КонецПериода,
        ,
        Счет = &СчетТоваров,
        КорСчет = &СчетРасчетов
    ) КАК ХозрасчетныйОбороты
```

#### Пример 3: Оборотно-сальдовая ведомость с корреспонденцией

### Важные моменты для регистров с корреспонденцией

1. **Используйте `В ИЕРАРХИИ`**: Для автоматической обработки корреспонденции всегда используйте условие `Счет В ИЕРАРХИИ (&Счет)` вместо прямого сравнения. Это позволяет получить данные не только по конкретному счету, но и по всем его субсчетам, а также автоматически обрабатывает обе стороны проводки.

2. **Поле `КорСчет`**: Автоматически показывает счет с противоположной стороны проводки. Если основной счет был в Дебете, то `КорСчет` будет `СчетКт`, и наоборот.

3. **Балансовые и небалансовые измерения**: В таблице итогов между счетами (`_AccRgCT<n>`) балансовые измерения и ресурсы хранятся в одном поле, а небалансовые - раздельно для Дт и Кт составляющих.

4. **Индексы**: Для регистров с корреспонденцией создаются отдельные индексы по `СчетДт` и `СчетКт`, что обеспечивает эффективный поиск по обеим сторонам проводки.

5. **Таблица `_AccRgCT<n>`**: Создается ТОЛЬКО для регистров, поддерживающих корреспонденцию, и используется для быстрого получения оборотов между конкретными парами счетов.

## Важные моменты

1. **Производительность**: Виртуальные таблицы остатков и оборотов значительно производительнее, чем выборка из таблицы движений с последующей группировкой. Платформа выполняет агрегирование на уровне СУБД.

2. **Условия отбора**: Условия по измерениям (Счет, Субконто и т.д.) лучше передавать в параметры виртуальной таблицы, а не в блок `ГДЕ`. Это позволяет платформе сузить выборку данных на более раннем этапе.

3. **Псевдонимы**: Используйте псевдонимы (`КАК <Псевдоним>`) для краткости и читаемости кода.

4. **Корреспонденция**: Для регистров с корреспонденцией используются поля `СчетДт`, `СчетКт`, `СубконтоДт`, `СубконтоКт` вместо `Счет` и `Субконто`. Виртуальные таблицы автоматически обрабатывают обе стороны проводки при использовании условия `Счет В ИЕРАРХИИ`.

5. **Виртуальные таблицы не хранятся в БД**: При обращении к виртуальным таблицам система автоматически собирает информацию из физических таблиц движений и итогов для выполнения запроса.
