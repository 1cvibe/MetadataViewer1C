# Таблицы запросов регистра накопления

## Виртуальные таблицы для запросов

Регистр накопления предоставляет четыре основные виртуальные таблицы:

1. **Движения** - возвращает все записи движений регистра за указанный период
2. **Остатки** - возвращает остатки (итоги) на конкретный момент времени
3. **Обороты** - возвращает обороты (приход, расход) за указанный период
4. **ОстаткиИОбороты** - универсальная таблица, объединяющая остатки на начало, обороты за период и остатки на конец

## Виды регистров накопления

Регистры накопления делятся на два типа:
- **Регистры остатков** - предназначены для хранения информации о наличии и движении ресурсов (остатки товаров, взаиморасчеты)
- **Регистры оборотов** - предназначены для хранения информации, для которой понятие остатка лишено смысла (объемы продаж, суммы закупок)

## Виртуальная таблица Движения

### Синтаксис
```
РегистрНакопления.<ИмяРегистра>.Движения(
    &НачалоПериода,
    &КонецПериода,
    СостояниеПериода = <Значение>,
    Условие = <Строка>
) КАК <Псевдоним>
```

### Параметры
- **Начало/Конец периода** - границы временного интервала для выборки движений
- **СостояниеПериода** - определяет, какие движения включать:
  - `ЕстьДвижения` - в периоде есть движения
  - `ЕстьДвиженияГраницыПериода` - на границе периода есть движения
- **Условие** - строковое выражение для фильтрации движений

### Доступные поля
- `Регистратор` - документ, создавший движение
- `Период` - момент времени движения
- `ВидДвижения` - Приход или Расход
- `НомерСтроки` - номер строки движения
- Все измерения, ресурсы и реквизиты регистра

### Пример
```sql
ВЫБРАТЬ
    ПродажиДвижения.Номенклатура,
    ПродажиДвижения.Период,
    ПродажиДвижения.Количество,
    ПродажиДвижения.Сумма
ИЗ
    РегистрНакопления.Продажи.Движения(
        &НачалоПериода,
        &КонецПериода,
        СостояниеПериода = ЕстьДвижения
    ) КАК ПродажиДвижения
ГДЕ
    ПродажиДвижения.Контрагент = &Контрагент
```

## Виртуальная таблица Остатки

### Синтаксис
```
РегистрНакопления.<ИмяРегистра>.Остатки(
    &Период,
    Условие = <Строка>
) КАК <Псевдоним>
```

### Параметры
- **Период** - дата, на которую нужно получить остатки (обязательный параметр)
- **Условие** - строковое выражение для фильтрации по измерениям и реквизитам

### Доступные поля
- `КоличествоОстаток`, `СуммаОстаток` - остатки по каждому ресурсу
- `КоличествоОстатокПриход`, `СуммаОстатокПриход` - остатки прихода
- `КоличествоОстатокРасход`, `СуммаОстатокРасход` - остатки расхода
- Все измерения регистра

### Пример
```sql
ВЫБРАТЬ
    ТоварыНаСкладахОстатки.Номенклатура,
    ТоварыНаСкладахОстатки.Склад,
    ТоварыНаСкладахОстатки.КоличествоОстаток,
    ТоварыНаСкладахОстатки.СуммаОстаток
ИЗ
    РегистрНакопления.ТоварыНаСкладах.Остатки(
        &Дата,
        Склад = &ВыбСклад
    ) КАК ТоварыНаСкладахОстатки
ГДЕ
    ТоварыНаСкладахОстатки.КоличествоОстаток > 0
```

## Виртуальная таблица Обороты

### Синтаксис
```
РегистрНакопления.<ИмяРегистра>.Обороты(
    &НачалоПериода,
    &КонецПериода,
    &Периодичность,
    Условие = <Строка>
) КАК <Псевдоним>
```

### Параметры
- **Начало/Конец периода** - границы временного интервала (обязательные)
- **Периодичность** - детализация разбивки итогов:
  - `Авто` (по умолчанию) - итоги не детализируются
  - `День` - итоги за каждый день
  - `Неделя` - итоги за каждую неделю
  - `Месяц` - итоги за каждый месяц
  - `Квартал` - итоги за каждый квартал
  - `Год` - итоги за каждый год
- **Условие** - строковое выражение для фильтрации

### Доступные поля
- `КоличествоПриход`, `СуммаПриход` - обороты по приходу
- `КоличествоРасход`, `СуммаРасход` - обороты по расходу
- `КоличествоОборот`, `СуммаОборот` - общие обороты (приход - расход)
- `Период` - период оборота (если указана периодичность, отличная от Авто)
- `ПериодДС` - дата конца периода (если указана периодичность)
- Все измерения регистра

### Пример
```sql
ВЫБРАТЬ
    ПродажиОбороты.Номенклатура,
    ПродажиОбороты.Период,
    ПродажиОбороты.КоличествоПриход,
    ПродажиОбороты.КоличествоРасход,
    ПродажиОбороты.СуммаОборот
ИЗ
    РегистрНакопления.Продажи.Обороты(
        &НачалоГода,
        &КонецГода,
        Месяц,
        Номенклатура = &ВыбНоменклатура
    ) КАК ПродажиОбороты
УПОРЯДОЧИТЬ ПО
    ПродажиОбороты.Период
```

## Виртуальная таблица ОстаткиИОбороты

### Синтаксис
```
РегистрНакопления.<ИмяРегистра>.ОстаткиИОбороты(
    &НачалоПериода,
    &КонецПериода,
    &Периодичность,
    Условие = <Строка>,
    СпособЗаполнения = <Строка>
) КАК <Псевдоним>
```

### Параметры
- **Начало/Конец периода** - границы временного интервала (обязательные)
- **Периодичность** - аналогично таблице `Обороты`:
  - `Авто`, `День`, `Неделя`, `Месяц`, `Квартал`, `Год`
- **Условие** - строковое выражение для фильтрации
- **СпособЗаполнения** (необязательный):
  - `"Движения"` - только поля оборотов
  - `"Остатки"` - только поля остатков
  - `"ОстаткиИОбороты"` (по умолчанию) - доступны и остатки, и обороты

### Доступные поля

**Для остатков:**
- `КоличествоНачальныйОстаток`, `СуммаНачальныйОстаток` - остатки на начало периода
- `КоличествоКонечныйОстаток`, `СуммаКонечныйОстаток` - остатки на конец периода
- `КоличествоНачальныйОстатокПриход`, `СуммаНачальныйОстатокПриход`
- `КоличествоНачальныйОстатокРасход`, `СуммаНачальныйОстатокРасход`
- `КоличествоКонечныйОстатокПриход`, `СуммаКонечныйОстатокПриход`
- `КоличествоКонечныйОстатокРасход`, `СуммаКонечныйОстатокРасход`

**Для оборотов:**
- `КоличествоПриход`, `СуммаПриход` - обороты по приходу
- `КоличествоРасход`, `СуммаРасход` - обороты по расходу
- `КоличествоОборот`, `СуммаОборот` - общие обороты

**Дополнительные поля:**
- `Период` - период оборота (если указана периодичность)
- `ПериодДС` - дата конца периода
- Все измерения регистра

### Пример: Оборотно-сальдовая ведомость по складам
```sql
ВЫБРАТЬ
    ТоварыОИО.Номенклатура,
    ТоварыОИО.Склад,
    ТоварыОИО.Период,
    ТоварыОИО.КоличествоНачальныйОстаток,
    ТоварыОИО.КоличествоПриход,
    ТоварыОИО.КоличествоРасход,
    ТоварыОИО.КоличествоКонечныйОстаток
ИЗ
    РегистрНакопления.ТоварыНаСкладах.ОстаткиИОбороты(
        &НачалоПериода,
        &КонецПериода,
        Месяц,
        Склад = &ВыбСклад
    ) КАК ТоварыОИО
УПОРЯДОЧИТЬ ПО
    ТоварыОИО.Номенклатура,
    ТоварыОИО.Период
```

## Физические таблицы базы данных

Для каждого регистра накопления создаются следующие физические таблицы:

### Таблица движений (_AccumRg<n>)

Содержит поля:
- `_Period` - момент времени движения
- `_Recorder<suff>` - ссылка на регистратор
- `_LineNo` - номер движения по данному регистратору
- `_Active` - признак того, что движение активно (участвует в подсчете остатков и оборотов)
- `_RecordKind` - вид движения (приход/расход)
- `_Fld<n><suff>` - измерение, ресурс или реквизит (для каждого измерения, каждого ресурса и каждого реквизита)
- `_DimHash` - хеш-функция измерений (присутствует, если количество измерений не позволяет организовать уникальный индекс)
- `_Fld<n>` - общие реквизиты

### Таблица остатков (_AccumRgT<n>)

**Создается, если регистр поддерживает остатки**

Содержит поля:
- `_Period` - момент времени, на который посчитаны остатки
- `_Fld<n><suff>` - измерение или остаток по ресурсу (для каждого измерения и каждого ресурса)
- `_Splitter` - разделитель итогов (для обеспечения параллельности проведения документов)
- `_DimHash` - хеш-функция измерений
- `_Fld<n>` - общие реквизиты

### Таблица оборотов (_AccumRgTn<n>)

**Создается, если регистр поддерживает обороты**

Содержит поля:
- `_Period` - момент времени, на который посчитаны обороты
- `_Fld<n><suff>` - измерение или оборот по ресурсу (для каждого измерения и каждого ресурса)
- `_Splitter` - разделитель итогов
- `_DimHash` - хеш-функция измерений
- `_Fld<n>` - общие реквизиты

### Таблица регистрации изменений (_AccumRgChngR<n>)

**Создается, если регистр участвует хотя бы в одном плане обмена**

Содержит поля аналогичные таблице регистрации изменений последовательности.

### Таблица настроек хранения итогов (_AccumRgOpt)

**Создается одна таблица на все регистры накопления**

Содержит поля:
- `_RegID` - идентификатор регистра
- `_Period` - периодичность хранения итогов (не используется)
- `_ActualPeriod` - хранение актуальных итогов
- `_Periodicity` - периодичность регистра
- `_RepetitionFactor` - кратность (не используется)
- `_UseTotals` - использовать итоги
- `_MinPeriod` - минимальный период, с которого нужно пересчитывать итоги
- `_MinCalculatedPeriod` - минимальный период, по которому нужно насчитывать итоги
- `_UseSplitter` - использовать разделитель итогов (для обеспечения параллельности)
- `_Fld<n>` - общие реквизиты

### Дополнительные таблицы для агрегатов

Для регистров накопления с агрегатами создаются дополнительные таблицы:
- `_AccumRgAgg<n>` - таблица агрегатов регистра накопления
- `_AccumRgAggOpt` - таблица опций сети агрегатов
- `_AccumRgSt<n>` - таблица статистики регистра накопления
- `_AccumRgBf<n>` - таблица буфера новых оборотов
- `_AccumRgDl<n>` - таблица новых оборотов
- `_AccumRgAggDims` - таблица кодов измерений регистра накопления
- `_AccumRgAggGrid` - таблица сети агрегатов

## Индексы таблицы движений

- **Кластерный индекс**: `Период + Измерение1 + ... + ИзмерениеN` (всегда)
- **Индекс по измерению**: `Измерение + Период + Регистратор + НомерСтроки` (если измерению задано свойство "Индексировать")
- **Индекс по реквизиту**: `Реквизит + Период + Регистратор + НомерСтроки` (если реквизиту задано свойство "Индексировать")

## Важные моменты

1. **Производительность**: Виртуальные таблицы остатков и оборотов значительно производительнее, чем выборка из таблицы движений с последующей группировкой. Платформа выполняет агрегирование на уровне СУБД, используя предрассчитанные итоги.

2. **Условия отбора**: Условия по измерениям лучше передавать в параметр `Условие` виртуальной таблицы, а не в блок `ГДЕ` запроса. Это позволяет платформе сузить выборку данных на более раннем этапе и использовать индексы эффективнее.

3. **Периодичность**: При указании периодичности, отличной от `Авто`, в результатах появляются поля `Период` и `ПериодДС`, что позволяет получать данные с детализацией по времени без дополнительных группировок.

4. **Разделитель итогов**: Механизм разделителя итогов (`_Splitter`) обеспечивает параллельность проведения документов. При одновременной записи движений несколькими сеансами система записывает изменения итогов отдельно, а при получении итогов эти данные складываются.

5. **Регистры остатков vs регистры оборотов**: Регистры остатков поддерживают получение остатков на момент времени, регистры оборотов - только обороты за период.

6. **Виртуальные таблицы не хранятся в БД**: При обращении к виртуальным таблицам система автоматически собирает информацию из физических таблиц движений и итогов для выполнения запроса.

7. **Использование параметров**: Всегда используйте параметры запроса (`&Дата`, `&НачалоПериода`) для передачи значений, это обеспечивает безопасность и производительность.
