# Обзор всех виртуальных таблиц запросов 1С

## Общие принципы

Виртуальные таблицы — это специальный механизм платформы 1С:Предприятие, который предоставляет готовые, предварительно обработанные срезы данных по регистрам. Они не хранятся физически в базе данных, а генерируются платформой "на лету" при выполнении запроса на основе физических таблиц движений и итогов.

**Преимущества виртуальных таблиц:**
- Высокая производительность (платформа оптимизирует запросы на уровне СУБД)
- Упрощение кода (не нужно вручную делать группировки и агрегацию)
- Автоматическое использование предрассчитанных итогов

## Классификация виртуальных таблиц по типам регистров

### 1. Регистры накопления

**Виртуальные таблицы:**
- `Движения` - все записи движений за период
- `Остатки` - остатки на конкретную дату
- `Обороты` - обороты за период
- `ОстаткиИОбороты` - универсальная таблица (остатки + обороты)

**Физические таблицы:**
- `_AccumRg<n>` - таблица движений
- `_AccumRgT<n>` - таблица итогов (для регистров остатков)
- `_AccumRgTn<n>` - таблица оборотов (для регистров оборотов)
- `_AccumRgOpt` - таблица настроек хранения итогов

### 2. Регистры бухгалтерии

**Виртуальные таблицы:**
- `Движения` - все проводки
- `Остатки` - сальдо на дату
- `Обороты` / `ОборотыД` - обороты за период (синонимы)
- `ОстаткиИОбороты` - оборотно-сальдовая ведомость
- `ДвиженияССубконто` - проводки с разверткой по субконто (специфично)

**Физические таблицы:**
- `_AccRg<n>` - таблица движений
- `_AccRgED<n>` - таблица значений субконто
- `_AccRgAT0<n>` - таблица итогов по счету
- `_AccRgAT<i><n>` - таблицы итогов по счету с субконто
- `_AccRgCT<n>` - таблица итогов оборотов между счетами (только для регистров с корреспонденцией)
- `_AccRgOpt` - таблица настроек хранения итогов

**Особенности:**
- Регистры могут поддерживать корреспонденцию счетов (СчетДт/СчетКт)
- Для регистров с корреспонденцией используется автоматическая обработка через условие `Счет В ИЕРАРХИИ`

### 3. Регистры расчета

**Виртуальные таблицы:**
- `Движения` - все записи расчета
- `ПериодДействия` - **специфичная таблица для регистров расчета**, разворачивает записи на временной шкале
- `СрезПоследних` - актуальные данные на дату
- `ДанныеГрафика` - данные из связанного регистра (графика)

**Физические таблицы:**
- `_CRg<n>` - таблица движений
- `_CRgActP<n>` - таблица фактических периодов действия (если установлен флаг "Период действия")
- `_CRgChngR<n>` - таблица регистрации изменений
- `_CRgRecalc<n>` - таблица перерасчета
- `_CRgRecalcChngR<n>` - таблица регистрации изменений перерасчета

**Особенности:**
- Понятие "период действия" и "период регистрации"
- Базовые измерения для индексации
- Механизм вытеснения расчетов

### 4. Регистры сведений

**Виртуальные таблицы:**
- `Движения` - все записи за период (для периодических регистров)
- `СрезПоследних` - последняя запись для каждой комбинации измерений на дату
- `СрезПервых` - первая запись для каждой комбинации измерений начиная с даты

**Физические таблицы:**
- `_InfoRg<n>` - таблица движений регистра сведений
- `_InfoRgSL<n>` - таблица срез последних (если включен признак РазрешитьИтогиСрезПоследних)
- `_InfoRgSF<n>` - таблица срез первых (если включен признак РазрешитьИтогиСрезПервых)
- `_InfoRChngRg<n>` - таблица регистрации изменений

**Особенности:**
- Могут быть периодическими и непериодическими
- Для периодических регистров используется механизм срезов

## Общие параметры виртуальных таблиц

### Параметры периода

- **`НачалоПериода`**, **`КонецПериода`** - границы временного интервала (для таблиц, работающих с периодами)
- **`Период`** - конкретная дата (для таблиц, работающих с моментами времени)

### Параметр Периодичность

Определяет детализацию разбивки итогов по времени:
- `Авто` - автоматический выбор, итоги не детализируются
- `РЕГИСТРАЦИЯ` / `Запись` - по каждой записи (построчно)
- `ДЕНЬ` / `День` - итоги за каждый день
- `НЕДЕЛЯ` / `Неделя` - итоги за каждую неделю
- `МЕСЯЦ` / `Месяц` - итоги за каждый месяц
- `КВАРТАЛ` / `Квартал` - итоги за каждый квартал
- `ГОД` / `Год` - итоги за каждый год

### Параметр Условие

Структура или строковое выражение для отбора записей:
- Отбор по измерениям и реквизитам
- Применяется до формирования виртуальной таблицы (для оптимизации)
- Лучше использовать в параметре `Условие`, чем в блоке `ГДЕ` запроса

### Параметр СостояниеПериода / СостояниеМомента

Определяет обработку границ периода:
- `ЕстьДвижения` - в периоде есть движения
- `ЕстьДвиженияГраницыПериода` - на границе периода есть движения
- `Без границ` - исключить границы периода
- `Включать границы` - включить границы периода
- `Включать первую границу` - включить только первую границу

## Рекомендации по использованию

1. **Всегда используйте виртуальные таблицы** вместо прямого обращения к таблицам движений для получения итогов
2. **Указывайте условия в параметрах виртуальной таблицы**, а не в блоке ГДЕ - это обеспечивает лучшую оптимизацию
3. **Используйте параметры запроса** для передачи дат и значений, а не подстановку значений напрямую
4. **Выбирайте правильную виртуальную таблицу** в зависимости от задачи:
   - Нужны только остатки → `Остатки`
   - Нужны только обороты → `Обороты`
   - Нужны и остатки, и обороты → `ОстаткиИОбороты`
5. **Для регистров расчета** используйте `ПериодДействия` для анализа данных во времени
6. **Для регистров сведений** используйте `СрезПоследних` для получения актуальных данных
