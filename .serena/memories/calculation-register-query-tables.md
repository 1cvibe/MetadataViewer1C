# Таблицы запросов регистра расчета

## Основные понятия

### Период регистрации
Дата начала периода, указанного при регистрации расчета (вычисляется по дате документа-регистратора). Например, в июне 2008 г. производится начисление оклада за май 2008 г. (расчеты с периодичностью Месяц). Май 2008 г. - период действия (в базе данных записывается дата 01.05.2008), а июнь 2008 г. - период регистрации (в базе данных записывается дата 01.06.2008).

### Период действия
Начальная дата периода, определяемая в соответствии со значением свойства Периодичность. Например, в документе указывается, что расчет производится за май 2008 г. Для значения свойства Периодичность регистра Месяц период действия определяется датой 01.05.2008; для значения Квартал - 01.04.2008.

### Период действия расчета
Указывает период, за который производится расчет. Период определяется датой начала и датой окончания периода. Например, запись о больничном листе за май 2008 г. имеет период действия 01.05.2008, а период действия расчета определяется датой начала (например, 06.05.2008) и датой окончания (например, 15.05.2008).

## Физические таблицы базы данных

Для каждого регистра расчета создаются следующие таблицы:

### Таблица движений (_CRg<n>)

Содержит поля:
- **`_Period`** - момент времени движения (период регистрации)
- **`_Recorder<suff>`** - ссылка на регистратор
- **`_LineNo`** - номер движения по данному регистратору
- **`_CalcKind<suff>`** - ссылка на вид расчета
- **`_ActionPeriod`** - период действия (присутствует, если в конфигурации у регистра установлен флаг "Период действия")
- **`_APDateFrom`** - момент начала периода действия (присутствует, если установлен флаг "Период действия")
- **`_APDateTill`** - момент окончания периода действия (присутствует, если установлен флаг "Период действия")
- **`_BPDateFrom`** - момент начала базового периода (присутствует, если установлен флаг "Базовый период")
- **`_BPDateTill`** - момент окончания базового периода (присутствует, если установлен флаг "Базовый период")
- **`_Active`** - признак того, что движение активно (участвует в расчете)
- **`_Storno`** - признак того, что движение является сторно-записью
- **`_Fld<n><suff>`** - измерение, ресурс или реквизит (для каждого измерения, каждого ресурса и каждого реквизита)
- **`_Fld<n>`** - общие реквизиты

### Индексы таблицы движений

- **Кластерный индекс**: `ПериодРегистрации + Регистратор + НомерСтроки` (всегда)
- **Индекс по базовым измерениям**: `ПериодРегистрации + [БазовоеИзмерение1 +...]` (всегда, по полю ПериодРегистрации и всем базовым измерениям)
- **Индекс по базовым измерениям (обратный)**: `[БазовоеИзмерение1 +...] + ПериодРегистрации` (если есть хоть одно базовое измерение)
- **Индекс по периоду действия**: `ПериодДействия + [БазовоеИзмерение1 +...]` (если установлено свойство регистра расчета "ПериодДействия")
- **Индекс по периоду действия (обратный)**: `[БазовоеИзмерение1 +...] + ПериодДействия` (если установлено свойство регистра расчета "ПериодДействия" и есть хоть одно базовое измерение)
- **Индекс по измерению**: `Измерение + ПериодРегистрации + Регистратор + НомерСтроки` (если измерению "Измерение" задано свойство "Индексировать")
- **Индекс по реквизиту**: `Реквизит + ПериодРегистрации + Регистратор + НомерСтроки` (если реквизиту "Реквизит" задано свойство "Индексировать")

### Таблица фактических периодов действия (_CRgActP<n>)

**Создается, если у регистра расчета установлен флаг "Период действия"**

Содержит поля:
- **`_Recorder<suff>`** - ссылка на регистратор
- **`_LineNo`** - номер движения по данному регистратору
- **`_APDateFrom`** - момент начала фактического периода действия
- **`_APDateTill`** - момент окончания фактического периода действия
- **`_Fld<n>`** - общие реквизиты

**Индексы таблицы фактических периодов действия:**
- `Регистратор + НомерСтроки` (всегда, включая поля независимых разделителей)

### Таблица регистрации изменений (_CRgChngR<n>)

**Создается, если регистр расчета участвует хотя бы в одном плане обмена**

Содержит те же поля, что и таблица регистрации изменений последовательности.

### Таблица перерасчета (_CRgRecalc<n>)

**Создается для каждого перерасчета**

Содержит поля:
- **`_Recorder<suff>`** - ссылка на регистратор
- **`_CalcKind<suff>`** - ссылка на вид расчета
- **`_Fld<n><suff>`** - измерение (для каждого измерения)
- **`_Fld<n>`** - общие реквизиты

**Индексы таблицы перерасчета:**
- `Регистратор + ВидРасчета + [Измерение1 + ...]` (всегда, по полю Регистратор, полю ВидРасчета и всем полям измерений перерасчета)

### Таблица регистрации изменений перерасчета (_CRgRecalcChngR<n>)

**Создается, если перерасчет участвует хотя бы в одном плане обмена**

Содержит те же поля, что и таблица регистрации изменений последовательности.

## Виртуальные таблицы для запросов

Регистр расчета предоставляет следующие виртуальные таблицы:

1. **Движения** - детальные данные регистра, содержащие все записи расчета
5. **ПериодДействия** - специфичная таблица для развертки записей на временной шкале
7. **ДанныеГрафика** - данные из связанного регистра расчета (графика)

### 1. Движения

Получение первичных данных для расчета - всех записей регистра расчета за указанный период.

**Синтаксис:**
```
РегистрРасчета.<ИмяРегистра>.Движения(
    &НачалоПериода,
    &КонецПериода,
    Условие = <Строка>
)
```

**Параметры:**
- **Начало/Конец периода** - период регистрации, за который нужно получить движения
- **Условие** - строковое выражение для фильтрации

**Доступные поля:**
- `Регистратор` - документ, создавший движение
- `Период` - период регистрации
- `ПериодДействия` - период действия (если установлен флаг "Период действия")
- `ВидРасчета` - вид расчета
- `БазовыйПериодНачало`, `БазовыйПериодКонец` - базовый период (если установлен флаг "Базовый период")
- Все измерения, ресурсы и реквизиты регистра

### 2. ПериодДействия

Самая специфичная и важная таблица для регистров расчета. Позволяет "развернуть" записи регистра на временной шкале, учитывая их фактический период действия, который может отличаться от периода регистрации.

**Синтаксис:**
```
РегистрРасчета.<ИмяРегистра>.ПериодДействия(
    &ПериодДействияНачало,
    &ПериодДействияКонец,
    Отбор = <Структура>,
    Условие = <Строка>,
    ТолькоДействующие = <Булево>,
    СпособРазложения = <Строка>,
    ВидДвижения = <Строка>
)
```

**Параметры:**
- **ПериодДействия** (обязательный) - интервал времени, на который "разворачиваются" периоды действия записей
- **Отбор** - структура отбора по измерениям и реквизитам
- **Условие** - строковое выражение для WHERE
- **ТолькоДействующие** (Булево) - если `Истина`, то в результат попадут только те "виртуальные" записи, для которых дата `ПериодДействия` попадает внутрь реального периода действия записи. Если `Ложь`, для записей, не действующих в указанный период, могут быть созданы строки с пустыми значениями. Чаще всего используется `Истина`
- **СпособРазложения** - определяет, на какой минимальный промежуток времени будет разложен период действия:
  - `"День"` (по умолчанию) - по каждой записи (построчно)
  - `"Неделя"` - итоги за каждую неделю
  - `"Месяц"` - итоги за каждый месяц
  - `"Квартал"` - итоги за каждый квартал
  - `"Год"` - итоги за каждый год
- **ВидДвижения** - позволяет отфильтровать только приходящие или только списываемые записи: `"Приход"` или `"Расход"`

**Поля, возвращаемые таблицей:**
- **`ПериодДействия`** - Дата (или начало периода в зависимости от способа разложения). Это то, для чего она и создана
- **`НомерЗаписи`** - Порядковый номер исходной записи в периоде (для упорядочивания)
- **`БазовыйПериодНачало`, `БазовыйПериодКонец`** - Период действия исходной "базовой" записи
- **`ВидДвиженияБазовый`** - Вид движения исходной записи (`Приход` или `Расход`)
- Все измерения, ресурсы и реквизиты регистра

**Пример: Рассчитать, сколько дней сотрудник находился в отпуске в январе 2024 года**
```sql
ВЫБРАТЬ
    РАЗЛИЧНЫЕ
    РРПериодДействия.ПериодДействия КАК ДеньОтпуска,
    РРПериодДействия.Сотрудник,
    РРПериодДействия.ВидРасчета
ИЗ
    РегистрРасчета.Начисления.ПериодДействия(
        &НачалоМесяца,
        &КонецМесяца,
        Отбор = Сотрудник = &Сотрудник,
        ТолькоДействующие = Истина,
        Условие = "ВидРасчета = &ВидРасчетаОтпуск"
    ) КАК РРПериодДействия
```

В этом запросе мы получим список всех дат января, на которые приходился отпуск. Количество строк в результате и будет искомым количеством дней.

### 6. ДанныеГрафика

Предоставляет данные из связанного регистра расчета (графика). Используется в запросах к регистру расчета для доступа к исходным данным, по которым производится расчет (например, график работы сотрудников).

**Синтаксис:**
```
РегистрРасчета.<ИмяРегистра>.ДанныеГрафика(
    &НачалоПериода,
    &КонецПериода,
    Условие = <Строка>
)
```

**Параметры:**
- **Начало/Конец периода** - период, за который нужно получить данные графика
- **Условие** - строковое выражение для фильтрации

**Доступные поля:**
- Поля из связанного регистра (графика), определенные в конфигурации

**Примечание:** Эта таблица доступна только если у регистра расчета настроена связь с другим регистром расчета (графиком) в конфигурации.

## Особенности регистров расчета

### Период действия vs Период регистрации

- **Период регистрации** - когда запись создана (дата документа-регистратора)
- **Период действия** - за какой период действует расчет (определяется периодичностью регистра)

### Период действия расчета

Может отличаться от периода действия регистра. Например, больничный лист за май может иметь период действия регистра 01.05.2008, но период действия расчета с 06.05.2008 по 15.05.2008.

### Базовые измерения

Измерения, у которых установлено свойство "Базовое". Используются для индексации и поиска связанных записей. Индексы создаются по комбинации `ПериодРегистрации` (или `ПериодДействия`) и базовых измерений.

### Активность движения

Поле `_Active` определяет, участвует ли запись в расчете. Только активные движения учитываются в виртуальных таблицах.

### Сторно

Поле `_Storno` указывает, является ли движение сторно-записью (отменяющей предыдущее движение).

### Таблица фактических периодов действия

Хранит рассчитанные фактически действующие периоды с учетом вытеснения и других механизмов конкуренции. Используется системой для оптимизации запросов к виртуальной таблице `ПериодДействия`.

### Перерасчеты

Подчиненные объекты регистра, которые позволяют задать правила взаимного влияния движений регистров. Хранятся в отдельной таблице `_CRgRecalc<n>`.

## Важные моменты и лучшие практики

1. **Производительность**: Виртуальные таблицы регистра расчета оптимизированы на уровне платформы. Использование прямого обращения к таблице движений `_CRg<n>` будет менее эффективным.

2. **Параметры периода действия**: При использовании `ОстаткиИОбороты` важно правильно указать `ПериодДействияНачало` и `ПериодДействияКонец`, чтобы получить корректные результаты.

3. **Разложение периодов**: Таблица `ПериодДействия` может генерировать большое количество строк при разложении на дни. Используйте `СпособРазложения = "Месяц"` или `"Квартал"` для агрегированных отчетов.

4. **ТолькоДействующие**: В большинстве случаев для `ПериодДействия` следует использовать `ТолькоДействующие = Истина`, чтобы получать только актуальные данные.

5. **Отбор vs Условие**: Условия по измерениям лучше передавать через параметр `Отбор`, а не через `Условие` в блоке `ГДЕ`. Это позволяет платформе оптимизировать запрос на более раннем этапе.

6. **Базовые измерения**: При проектировании регистра расчета правильно определите базовые измерения (например, Сотрудник), чтобы обеспечить эффективную индексацию и быстрый поиск связанных записей.

7. **Ограничение памяти**: При расчете фактического периода действия, который выполняется при каждой записи набора записей регистра, может возникнуть ошибка нехватки памяти в том случае, если в расчете участвует большое число одинаковых записей. Это ограничение обычно не проявляется в условиях реальной практики.
