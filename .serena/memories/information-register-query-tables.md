# Таблицы запросов регистра сведений

## Общая информация

Регистр сведений предназначен для хранения разрозненной информации в разрезе конкретных значений (измерений) и, опционально, во времени (периодические регистры сведений).

**Периодический регистр сведений** - регистр, который для каждой уникальной комбинации измерений может хранить несколько записей, но актуальной является та, у которой дата наиболее близка (но не превышает) к моменту времени, к которому делается запрос.

## Виртуальные таблицы для запросов

Регистр сведений предоставляет три основные виртуальные таблицы:

1. **Движения** - все записи регистра за указанный период (для периодических регистров)
2. **СрезПоследних** - последняя по дате запись для каждой комбинации измерений на указанный момент времени
3. **СрезПервых** - первая по дате запись для каждой комбинации измерений начиная с указанной даты

## Виртуальная таблица Движения

### Назначение
Получение всех записей регистра сведений за указанный период. Используется для периодических регистров сведений.

### Синтаксис
```
РегистрСведений.<ИмяРегистра>.Движения(
    &НачалоПериода,
    &КонецПериода,
    Условие = <Строка>
) КАК <Псевдоним>
```

### Параметры
- **Начало/Конец периода** - границы временного интервала для выборки записей
- **Условие** - строковое выражение для фильтрации по измерениям и реквизитам

### Доступные поля
- `Период` - момент времени записи (присутствует для периодических регистров)
- `Регистратор` - ссылка на регистратор (если режим записи "Подчинение регистратору")
- `НомерСтроки` - номер строки (если режим записи "Подчинение регистратору")
- `Активность` - признак активности записи
- Все измерения, ресурсы и реквизиты регистра

### Пример
```sql
ВЫБРАТЬ
    ЦеныНоменклатурыДвижения.Номенклатура,
    ЦеныНоменклатурыДвижения.ТипЦен,
    ЦеныНоменклатурыДвижения.Период,
    ЦеныНоменклатурыДвижения.Цена
ИЗ
    РегистрСведений.ЦеныНоменклатуры.Движения(
        &НачалоПериода,
        &КонецПериода,
        Номенклатура = &ВыбНоменклатура
    ) КАК ЦеныНоменклатурыДвижения
УПОРЯДОЧИТЬ ПО
    ЦеныНоменклатурыДвижения.Период УБЫВ
```

## Виртуальная таблица СрезПоследних

### Назначение
Получить **последнюю** по дате версию записи для каждой комбинации измерений на указанный момент времени. Это наиболее часто используемая таблица для получения актуальных данных (курсы валют, цены номенклатуры и т.д.).

### Синтаксис
```
РегистрСведений.<ИмяРегистра>.СрезПоследних(
    &Период,
    Условие = <Структура или Строка>
) КАК <Псевдоним>
```

### Параметры
- **Период** (обязательный) - момент времени, на который необходимо получить актуальные данные. Платформа найдет последнюю запись, у которой поле `Период` меньше или равно этому значению.
  - Тип: `Дата`, `Граница`, `СтандартныйПериод`
  - Примеры: `&ТекущаяДата`, `'20230930'`, `Граница`
- **Условие** (необязательный) - отбор по измерениям и ресурсам регистра **до** формирования среза:
  - Тип: `Структура` или `Строка`
  - Синтаксис структуры: `Новый Структура("ИмяИзмерения1, ИмяИзмерения2", Значение1, Значение2)`
  - Синтаксис строки: `"Номенклатура = &Номенклатура И ТипЦен = &ТипЦен"`
  - **Важно**: Отбор в параметре `Условие` выполняется на уровне СУБД над всеми записями регистра, что обеспечивает высокую производительность

### Доступные поля
- Все измерения, ресурсы и реквизиты регистра
- `Период` - период актуальной записи

### Пример: Получить актуальные цены на номенклатуру
```sql
ВЫБРАТЬ
    ЦеныНоменклатурыСрезПоследних.Номенклатура,
    ЦеныНоменклатурыСрезПоследних.ТипЦен,
    ЦеныНоменклатурыСрезПоследних.Цена,
    ЦеныНоменклатурыСрезПоследних.Период КАК ДатаУстановкиЦены
ИЗ
    РегистрСведений.ЦеныНоменклатуры.СрезПоследних(
        &ТекущаяДата,
        Новый Структура("Номенклатура", &НужнаяНоменклатура)
    ) КАК ЦеныНоменклатурыСрезПоследних
```

### Пример с использованием строкового условия
```sql
ВЫБРАТЬ
    КурсыВалютСрезПоследних.Валюта,
    КурсыВалютСрезПоследних.Курс,
    КурсыВалютСрезПоследних.Кратность
ИЗ
    РегистрСведений.КурсыВалют.СрезПоследних(
        &ТекущаяДата,
        "Валюта = &ВыбВалюта"
    ) КАК КурсыВалютСрезПоследних
```

## Виртуальная таблица СрезПервых

### Назначение
Получить **первую** по дате версию записи для каждой комбинации измерений, начиная с указанной даты. Используется реже, чем `СрезПоследних`, но незаменима для исторического анализа и получения начальных данных.

### Синтаксис
```
РегистрСведений.<ИмяРегистра>.СрезПервых(
    &Период,
    Условие = <Структура или Строка>
) КАК <Псевдоним>
```

### Параметры
- **Период** (обязательный) - момент времени, **начиная с которого** нужно искать записи. Платформа найдет самую раннюю запись, у которой поле `Период` больше или равно этому значению.
- **Условие** (необязательный) - аналогично `СрезПоследних`, отбор по измерениям и ресурсам

### Доступные поля
- Все измерения, ресурсы и реквизиты регистра
- `Период` - период первой записи

### Пример: Получить первые установленные цены с начала года
```sql
ВЫБРАТЬ
    ЦеныНоменклатурыСрезПервых.Номенклатура,
    ЦеныНоменклатурыСрезПервых.ТипЦен,
    ЦеныНоменклатурыСрезПервых.Цена КАК НачальнаяЦенаГода,
    ЦеныНоменклатурыСрезПервых.Период КАК ДатаУстановки
ИЗ
    РегистрСведений.ЦеныНоменклатуры.СрезПервых(
        '20230101',  // Начало года
        Новый Структура("ТипЦен", &ОсновнойТипЦен)
    ) КАК ЦеныНоменклатурыСрезПервых
```

## СрезПоследних и СрезПервых в разрезе периода

Эти виртуальные таблицы могут получать срез не на точную дату, а на начало каждого периода в указанном диапазоне.

### Синтаксис для периодических срезов
```
РегистрСведений.<ИмяРегистра>.СрезПоследних(
    &НачалоПериода,
    &КонецПериода,
    &Периодичность,
    Условие = <Строка>
) КАК <Псевдоним>
```

### Дополнительные параметры
- **Начало/Конец периода** - интервал времени для построения динамических рядов данных
- **Периодичность** - детализация разбивки:
  - `День` - срез на начало каждого дня
  - `Неделя` - срез на начало каждой недели
  - `Месяц` - срез на начало каждого месяца
  - `Квартал` - срез на начало каждого квартала
  - `Год` - срез на начало каждого года

### Пример: Динамика изменения цены по месяцам
```sql
ВЫБРАТЬ
    ЦеныНоменклатурыСрезПоследних.Номенклатура,
    ЦеныНоменклатурыСрезПоследних.Период,
    ЦеныНоменклатурыСрезПоследних.Цена
ИЗ
    РегистрСведений.ЦеныНоменклатуры.СрезПоследних(
        &НачалоГода,
        &КонецГода,
        Месяц,
        "Номенклатура = &ВыбНоменклатура"
    ) КАК ЦеныНоменклатурыСрезПоследних
УПОРЯДОЧИТЬ ПО
    ЦеныНоменклатурыСрезПоследних.Период
```

## Физические таблицы базы данных

Для каждого регистра сведений создаются следующие физические таблицы:

### Таблица движений (_InfoRg<n>)

Содержит поля:
- `_Period` - момент времени движения (присутствует, если значение свойства "Периодичность" отлично от "Непериодический")
- `_Recorder<suff>` - ссылка на регистратор (присутствует, если значением свойства "Режим записи" является "Подчинение регистратору")
- `_LineNo` - номер движения по данному регистратору (присутствует, если режим записи "Подчинение регистратору")
- `_Active` - движение активно (участвует в вычислении среза последних). Присутствует, если режим записи "Подчинение регистратору"
- `_Fld<n><suff>` - измерение, ресурс или реквизит (для каждого измерения, каждого ресурса и каждого реквизита)
- `_Fld<n>` - общие реквизиты

### Таблица срез последних (_InfoRgSL<n>)

**Создается для каждого периодического регистра сведений с включенным признаком РазрешитьИтогиСрезПоследних**

Содержит поля:
- `_Period` - момент времени движения
- `_Recorder<suff>` - ссылка на регистратор (присутствует, если режим записи "Подчинение регистратору")
- `_Fld<n><suff>` - измерение, ресурс или реквизит (для каждого измерения, каждого ресурса и каждого реквизита)

Эта таблица используется для оптимизации запросов к виртуальной таблице `СрезПоследних`.

### Таблица срез первых (_InfoRgSF<n>)

**Создается для каждого периодического регистра сведений с включенным признаком РазрешитьИтогиСрезПервых**

Содержит поля аналогично `_InfoRgSL<n>`. Используется для оптимизации запросов к виртуальной таблице `СрезПервых`.

### Таблица регистрации изменений (_InfoRChngRg<n>)

**Создается, если регистр сведений участвует хотя бы в одном плане обмена**

Содержит поля аналогичные таблице регистрации изменений последовательности.

## Индексы таблицы движений

- **Индекс по периоду**: `Период + Измерение1 + ... + ИзмерениеN` (для периодических регистров)
- **Индекс по измерению**: `Измерение + Период` (если измерению задано свойство "Индексировать")
- **Индекс по реквизиту**: `Реквизит + Период` (если реквизиту задано свойство "Индексировать")

## Особенности работы с регистрами сведений

### Периодичность регистра

В настройках регистра сведений можно указать "периодичность" (например, `День`, `Месяц`). Эта настройка **не влияет** на работу виртуальных таблиц `СрезПоследних` и `СрезПервых`. Они всегда работают с конкретной датой и временем записи из поля `Период`.

**Для чего нужна периодичность:**
- Используется механизмом управляемых/автономных периодов
- Контроль уникальности записей на уровне платформы
- Для периодичности `День` можно не вводить время, платформа автоматически подставит `00:00:00`

### Режимы записи

- **Автономный** - записи не привязаны к документам
- **Подчинение регистратору** - записи привязаны к документам-регистраторам, создаются поля `Регистратор`, `НомерСтроки`, `Активность`

### РазрешитьИтогиСрезПоследних / РазрешитьИтогиСрезПервых

Эти настройки включают предрасчет итогов для оптимизации запросов:
- `РазрешитьИтогиСрезПоследних` - создается таблица `_InfoRgSL<n>`
- `РазрешитьИтогиСрезПервых` - создается таблица `_InfoRgSF<n>`

**Важно**: Включение этих настроек ускоряет запросы к соответствующим виртуальным таблицам, но требует дополнительного места в базе данных и времени на пересчет при изменении данных.

## Важные моменты и лучшие практики

1. **Производительность**: Всегда передавайте отбор по измерениям в параметр `Условие` виртуальной таблицы, а не в блок `ГДЕ` запроса. Это позволяет платформе оптимизировать запрос и использовать предрассчитанные итоги из таблиц `_InfoRgSL<n>` и `_InfoRgSF<n>`.

2. **Использование структуры vs строки**: Для простых условий можно использовать строковое выражение, для сложных логических условий лучше использовать структуру с явным указанием полей.

3. **СрезПоследних vs СрезПервых**: 
   - `СрезПоследних` - 95% случаев работы с периодическими регистрами (актуальные данные на дату)
   - `СрезПервых` - для исторического анализа, получения начальных значений

4. **Всегда передавайте дату**: Параметр `Период` обязателен, это делает запрос детерминированным и позволяет платформе использовать оптимизации.

5. **Непериодические регистры**: Для непериодических регистров сведений виртуальные таблицы `СрезПоследних` и `СрезПервых` не имеют смысла, так как для каждой комбинации измерений может быть только одна запись. В этом случае используйте прямую выборку из таблицы движений.

6. **Периодические срезы**: Используйте параметры `НачалоПериода`, `КонецПериода`, `Периодичность` для построения динамических рядов данных (например, как менялась цена на товар в течение месяца).

7. **Виртуальные таблицы не хранятся в БД**: При обращении к виртуальным таблицам система автоматически собирает информацию из физических таблиц движений и итоговых таблиц для выполнения запроса.
