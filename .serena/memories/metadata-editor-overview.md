# Обзор редактора метаданных

## Архитектура

### Основные компоненты

1. **MetadataPanel.ts** (`src/panels/MetadataPanel.ts`)
   - Класс для управления webview панелью редактора метаданных
   - Открывает редактор для объектов метаданных и форм
   - Обрабатывает сохранение изменений
   - Сканирует метаданные конфигурации для автодополнения

2. **MetadataEditor.tsx** (`src/webview/components/MetadataEditor.tsx`)
   - Основной React компонент редактора
   - Управляет состоянием: объекты, выбранный объект, форма, XML
   - Синхронизирует изменения между формой и XML редактором
   - Управляет вкладками: properties, attributes, tabular, forms, commands, xml

3. **FormEditor.tsx** (`src/webview/components/FormEditor.tsx`)
   - Компонент формы на основе JSON Schema Form (@rjsf/core)
   - Редактирование свойств объектов метаданных
   - Управление реквизитами, табличными частями, формами, командами
   - Использует кастомные виджеты: TypeWidget, MultilingualWidget

4. **XmlEditor.tsx** (`src/webview/components/XmlEditor.tsx`)
   - Редактор XML с подсветкой синтаксиса (Monaco Editor)
   - Показывает исходный XML объекта
   - Позволяет редактировать XML напрямую

## Парсинг метаданных

### metadataParser.ts (`src/xmlParsers/metadataParser.ts`)

**Интерфейсы:**
- `ParsedMetadataObject` - основной интерфейс объекта метаданных
  - `objectType`: тип объекта (Catalog, Document, etc.)
  - `name`: имя объекта
  - `sourcePath`: путь к исходному XML файлу
  - `properties`: свойства объекта (Record<string, any>)
  - `attributes`: массив реквизитов (ParsedAttribute[])
  - `tabularSections`: массив табличных частей (ParsedTabularSection[])
  - `forms`: массив форм (ParsedForm[])
  - `commands`: массив команд (ParsedCommand[])
  - `predefined`: массив предопределенных элементов (ParsedPredefined[])
  - `_originalXml`: исходный XML как строка (для сохранения структуры)

- `ParsedAttribute` - реквизит
  - `uuid`: UUID из XML (для точной идентификации)
  - `childObjectKind`: тип дочернего объекта ('Attribute' | 'Resource' | 'Dimension')
  - `name`: имя реквизита
  - `type`: тип реквизита (ParsedTypeRef)
  - `typeDisplay`: отображаемый тип
  - `properties`: свойства реквизита

- `ParsedTabularSection` - табличная часть
  - `uuid`: UUID из XML
  - `name`: имя табличной части
  - `attributes`: массив реквизитов табличной части

**Функции:**
- `parseMetadataXml(xmlPath: string)`: парсинг XML файла метаданных
  - Сохраняет исходный XML в `_originalXml` для сохранения структуры
  - Использует `createXMLParser()` для парсинга
  - Извлекает свойства, реквизиты, табличные части, формы, команды

## Сохранение изменений

### Процесс сохранения

1. **Получение изменений из webview**
   - Webview отправляет сообщение `saveCurrent` с обновленным `ParsedMetadataObject`
   - `MetadataPanel.handleSave()` обрабатывает сохранение

2. **Применение изменений к XML**
   - Используется `applyChangesToXmlString()` для применения изменений к исходному XML
   - Сохраняется структура XML (атрибуты vs элементы)
   - Обновляется `ConfigDumpInfo.xml` для реквизитов/табличных частей

3. **Валидация и нормализация**
   - `normalizeXML()` - нормализация XML
   - `validateXML()` - валидация XML

4. **Создание резервной копии**
   - `createBackup()` - создание резервной копии перед сохранением

5. **Запись в файл**
   - `safeWriteFile()` - безопасная запись файла

### Ключевые функции сохранения

- `applyChangesToXmlString(originalXml, obj, xmlObjectType)`: применяет изменения к исходному XML через строковые замены
- `patchConfigDumpInfoForChildObjects()`: обновляет ConfigDumpInfo.xml для реквизитов/табличных частей
- `mergeWithOriginalStructure()`: объединяет изменения с исходной структурой XML

## Сканирование метаданных

### scanMetadataForWebview()

Сканирует конфигурацию для получения:
- **registers**: список регистров (InformationRegister, AccumulationRegister, etc.)
- **referenceTypes**: список ссылочных типов (CatalogRef, DocumentRef, etc.)

Используется для:
- Автодополнения в редакторе типов (TypeWidget)
- Валидации типов реквизитов

## Открытие редактора

### Методы открытия

1. **createOrShowForFile(extensionUri, filePath)**
   - Открывает редактор для конкретного файла
   - Определяет, является ли файл формой или объектом метаданных
   - Парсит файл и создает панель

2. **createOrShowFromTreeItem(extensionUri, item)**
   - Открывает редактор из дерева метаданных
   - Определяет путь к файлу на основе TreeItem
   - Проверяет существование файла

## Вкладки редактора

1. **properties** - Свойства объекта
   - Редактирование основных свойств через JSON Schema Form
   - Использует кастомные виджеты для типов и многоязычных полей

2. **attributes** - Реквизиты
   - Список реквизитов объекта
   - Добавление/редактирование/удаление реквизитов
   - Редактор типов для каждого реквизита

3. **tabular** - Табличные части
   - Список табличных частей
   - Добавление/редактирование/удаление табличных частей
   - Редактирование реквизитов табличных частей

4. **forms** - Формы
   - Список форм объекта
   - Открытие форм в редакторе форм

5. **commands** - Команды
   - Список команд объекта
   - Редактирование свойств команд

6. **xml** - XML редактор
   - Прямое редактирование XML
   - Показывает исходный XML объекта

## Особенности

### Сохранение структуры XML

- Используется `_originalXml` для сохранения исходной структуры
- Применяются точечные изменения через строковые замены
- Сохраняется формат (атрибут vs элемент) для неизмененных свойств

### UUID для идентификации

- Реквизиты и табличные части имеют UUID из XML
- UUID используется для точной идентификации при сохранении/удалении
- Обновляется ConfigDumpInfo.xml на основе UUID

### Поддержка форм

- Редактор может открывать как объекты метаданных, так и формы
- Формы парсятся через `parseFormXml()`
- Формы имеют свою структуру (атрибуты, команды)

### Многоязычность

- Поддержка многоязычных полей (Synonym, Comment)
- Используется виджет MultilingualWidget
- Сохраняется структура v8:item для многоязычных значений

## Интеграция с деревом метаданных

- Редактор открывается из дерева метаданных через контекстное меню
- Команда: `metadataViewer.openMetadataEditor`
- Определяет тип объекта и путь к файлу автоматически
