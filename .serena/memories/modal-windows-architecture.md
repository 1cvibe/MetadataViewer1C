# Архитектура модальных окон в проекте MetaDataViewer_v3

## Важное правило

**Модальные окна реализуются ТОЛЬКО через отдельные модули (React компоненты).**

## Запрещено

- ❌ Статические HTML элементы модальных окон в HTML шаблонах
- ❌ Инлайн модальные окна в JSX
- ❌ Прямое манипулирование DOM для модальных окон

## Разрешенный подход

- ✅ Отдельные React компоненты для каждого типа модального окна
- ✅ Базовый компонент `Modal.tsx` для общей структуры
- ✅ Модальные окна рендерятся через React, управляются через props `isOpen`

## Структура модальных окон

Все модальные окна находятся в:
- `src/webview/components/FormEditor/` - для FormEditor
- `src/webview/components/FormPreview/modals/` - для FormPreviewApp

### Базовый компонент
- `Modal.tsx` - базовый компонент модального окна с проверкой `if (!isOpen) return null;`

### Компоненты модальных окон для FormEditor
- `AttributeTypeEditorModal.tsx` - редактирование типа реквизита
- `AddAttributeToObjectModal.tsx` - добавление реквизита/ресурса/измерения
- `AddTabularSectionModal.tsx` - добавление табличной части
- `AddTabularAttributeModal.tsx` - добавление реквизита табличной части
- `EditTabularAttributeTypeModal.tsx` - редактирование типа реквизита табличной части
- `RegisterRecordsEditorModal.tsx` - редактирование регистров
- `ConfirmModal.tsx` - подтверждение действий

### Компоненты модальных окон для FormPreview
- `ConfirmDeleteModal.tsx` - подтверждение удаления
- `EditAttributeModal.tsx` - редактирование реквизита формы
- `EditColumnModal.tsx` - редактирование колонки
- `EditCommandModal.tsx` - редактирование команды

## Принципы

1. Каждое модальное окно - отдельный React компонент
2. Видимость контролируется через prop `isOpen`
3. Базовый компонент `Modal` возвращает `null` при `isOpen={false}`
4. Все модальные окна рендерятся в конце родительского компонента, вне условных блоков
5. Данные передаются через props, события через callbacks

## История

Статические HTML модальные окна были удалены из `MetadataPanel.ts` в пользу React компонентов для упрощения архитектуры и поддержки кода.
