# Чеклист завершения задачи в проекте MetaDataViewer_v3

## Перед завершением задачи

### 1. Компиляция и сборка
- [ ] Выполнить `npm run compile` - проверить отсутствие ошибок TypeScript
- [ ] Выполнить `npm run build:webview` - проверить сборку webview
- [ ] Убедиться, что нет ошибок компиляции

### 2. Линтинг
- [ ] Выполнить `npm run lint` - исправить все предупреждения линтера
- [ ] Проверить соответствие стилю кода проекта

### 3. Тестирование
- [ ] Протестировать изменения в режиме отладки (F5)
- [ ] Проверить работу с реальными XML файлами метаданных
- [ ] Убедиться, что сохранение XML работает корректно
- [ ] Проверить, что структура XML сохраняется (атрибуты vs элементы)

### 4. Проверка функциональности
- [ ] Убедиться, что изменения не ломают существующий функционал
- [ ] Проверить работу с разными типами объектов (Document, Catalog, Enum, etc.)
- [ ] Проверить работу с большими конфигурациями

### 5. Документация
- [ ] Обновить комментарии в коде, если изменилась логика
- [ ] Обновить README.md, если добавлена новая функциональность

### 6. Git
- [ ] Проверить статус изменений: `git status`
- [ ] Просмотреть изменения: `git diff`
- [ ] Убедиться, что не добавлены лишние файлы (node_modules, out/, etc.)

## Особые проверки для редактора метаданных

### Проверка сохранения XML структуры
- [ ] Убедиться, что свойства сохраняются как элементы, а не атрибуты
- [ ] Проверить сохранение многоязычных полей (v8:item структура)
- [ ] Проверить сохранение StandardAttributes
- [ ] Проверить сохранение InternalInfo (xr:TypeId, xr:ValueId как элементы)
- [ ] Проверить сохранение Type структур (OneOf, TypeSet, StringQualifiers)

### Проверка React компонентов
- [ ] Убедиться, что все хуки вызываются в правильном порядке
- [ ] Проверить отсутствие ошибок в консоли браузера
- [ ] Проверить работу табов и переключения между разделами
- [ ] Проверить синхронизацию между формой и XML редактором

## Команды для финальной проверки

```bash
# Полная проверка перед коммитом
npm run compile && npm run build:webview && npm run lint
```

## После завершения задачи

1. Создать коммит с описанием изменений
2. Если необходимо, обновить версию в `package.json`
3. Если изменения критичны, протестировать на нескольких конфигурациях 1С
