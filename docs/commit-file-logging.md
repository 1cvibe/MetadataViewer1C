# Автоматическое логирование изменений в Commit.txt

## Описание

Расширение автоматически отслеживает изменения файлов проекта и записывает их в файл `Commit.txt`. Это позволяет легко отслеживать, какие файлы были изменены в ходе работы.

## Настройка

### Путь к файлу Commit.txt

По умолчанию файл создается по пути: `${workspaceFolder}\build\commit.txt`

Вы можете изменить путь в настройках расширения:
1. Откройте настройки VS Code (Ctrl+,)
2. Найдите "1C Metadata Viewer"
3. Измените параметр "Commit File Path"

Поддерживается переменная `${workspaceFolder}` для указания относительного пути от корня проекта.

### Режим отладки

Для просмотра подробных логов работы системы логирования включите режим отладки:
1. Откройте настройки VS Code
2. Найдите "1C Metadata Viewer: Debug Mode"
3. Установите значение `true`

Логи будут отображаться в панели Output (канал "1C Metadata Viewer").

## Формат файла Commit.txt

Файл содержит список измененных файлов, по одному на строку:
- Полные пути к файлам
- Кодировка UTF-8
- Разделитель строк: `\r\n` (Windows)
- Строки, начинающиеся с `REM`, игнорируются (комментарии)
- Пустые строки не поддерживаются
- Дубликаты автоматически фильтруются

### Пример содержимого

```
d:\1C\RZDZUP\src\cf\Catalogs\ibs_ВариантыОтчетовДляКонсолидации.xml
d:\1C\RZDZUP\src\cf\Documents\ibs_ЗаданиеНаКонсолидацию.xml
d:\1C\RZDZUP\src\cf\CommonModules\ibs_КонсолидацияДанных\Ext\Module.bsl
REM Это комментарий - будет пропущен
d:\1C\RZDZUP\src\cf\Reports\ibs_ОтчетПоКонсолидации\Templates\ОсновнаяСхемаКомпоновкиДанных\Template.xml
```

## Отслеживаемые изменения

Система автоматически логирует изменения в следующих случаях:

### 1. Редактор метаданных (Edit Metadata)
При сохранении объекта метаданных через встроенный редактор:
- Справочники, Документы, Регистры и т.д.
- Изменения реквизитов, табличных частей
- Изменения свойств объекта

**Файлы:** `*.xml` (файлы объектов метаданных)

### 2. Редактор СКД (DCS Editor)
При сохранении схемы компоновки данных:
- Изменения наборов данных
- Изменения вычисляемых полей
- Изменения настроек отчета

**Файлы:** `Template.xml` (схемы компоновки данных)

### 3. Редактор форм (Form Editor)
При сохранении формы через редактор метаданных:
- Изменения реквизитов формы
- Изменения команд формы
- Изменения элементов формы

**Файлы:** `Form.xml` (файлы форм)

### 4. BSL модули
При сохранении любого BSL файла в редакторе VS Code:
- Модули объектов
- Модули менеджеров
- Модули форм
- Общие модули
- Модули команд
- И любые другие `.bsl` файлы

**Файлы:** `*.bsl` (все BSL модули)

## Технические детали

### Класс CommitFileLogger

Основной класс для работы с логированием: `src/utils/commitFileLogger.ts`

#### Методы

**`getInstance()`** - Получить экземпляр синглтона
```typescript
const logger = CommitFileLogger.getInstance();
```

**`logChangedFile(filePath: string)`** - Записать один файл
```typescript
logger.logChangedFile('d:\\1C\\RZDZUP\\src\\cf\\Catalogs\\ibs_ВариантыОтчетовДляКонсолидации.xml');
```

**`logChangedFiles(filePaths: string[])`** - Записать несколько файлов
```typescript
logger.logChangedFiles([
  'd:\\1C\\RZDZUP\\src\\cf\\Catalogs\\ibs_ВариантыОтчетовДляКонсолидации.xml',
  'd:\\1C\\RZDZUP\\src\\cf\\Documents\\ibs_ЗаданиеНаКонсолидацию.xml'
]);
```

### Интеграция

Логирование интегрировано в следующие компоненты:

1. **MetadataPanel** (`src/panels/MetadataPanel.ts`)
   - После успешного сохранения объекта метаданных
   - После успешного сохранения формы

2. **DcsEditor** (`src/dcsEditor.ts`)
   - После успешного сохранения Template.xml

3. **Extension** (`src/extension.ts`)
   - Обработчик события `onDidSaveTextDocument` для BSL файлов

### Обработка ошибок

- Все ошибки логируются в Output канал
- При ошибке записи показывается предупреждение пользователю
- Ошибки не блокируют основной процесс сохранения

### Производительность

- Файл читается только при необходимости проверки дубликатов
- Используется Set для быстрой проверки существования записей
- Запись выполняется в режиме append (добавление в конец файла)

## Использование в CI/CD

Файл `Commit.txt` может быть использован в автоматизированных процессах:

1. **Выборочная загрузка в конфигурацию**
   - Загружать только измененные файлы
   - Ускорение процесса сборки

2. **Анализ изменений**
   - Определение затронутых подсистем
   - Автоматическое тестирование только измененных модулей

3. **Документирование**
   - Автоматическое создание списка изменений
   - Формирование release notes

## Примеры использования

### Пример 1: Очистка списка перед началом работы

```powershell
# Удалить существующий файл
Remove-Item "build\commit.txt" -ErrorAction SilentlyContinue

# Или очистить содержимое
Clear-Content "build\commit.txt"
```

### Пример 2: Обработка списка измененных файлов

```powershell
# Прочитать список файлов
$files = Get-Content "build\commit.txt" | Where-Object { 
    $_ -and -not $_.StartsWith("REM") 
}

# Вывести количество измененных файлов
Write-Host "Изменено файлов: $($files.Count)"

# Обработать каждый файл
foreach ($file in $files) {
    Write-Host "Обработка: $file"
    # Ваша логика обработки
}
```

### Пример 3: Фильтрация по типу файлов

```powershell
$commitFile = "build\commit.txt"
$files = Get-Content $commitFile | Where-Object { 
    $_ -and -not $_.StartsWith("REM") 
}

# Только XML файлы метаданных
$xmlFiles = $files | Where-Object { $_ -match '\.xml$' }

# Только BSL модули
$bslFiles = $files | Where-Object { $_ -match '\.bsl$' }

Write-Host "XML файлов: $($xmlFiles.Count)"
Write-Host "BSL файлов: $($bslFiles.Count)"
```

## Ограничения

1. Файл создается только при первом изменении (не создается автоматически при старте)
2. Не отслеживаются изменения, сделанные вне VS Code
3. Не отслеживаются удаления файлов
4. Пути всегда нормализуются к Windows формату (обратные слеши)

## Устранение неполадок

### Файл не создается

1. Проверьте путь в настройках
2. Убедитесь, что директория существует (создается автоматически)
3. Проверьте права доступа к директории
4. Включите Debug Mode и проверьте логи

### Дубликаты в файле

Это не должно происходить - система автоматически фильтрует дубликаты.
Если дубликаты появляются:
1. Проверьте, что используется последняя версия расширения
2. Включите Debug Mode
3. Сообщите о проблеме разработчикам

### Неправильные пути

Пути нормализуются автоматически:
- Прямые слеши заменяются на обратные
- Убираются лишние слеши
- Пути приводятся к абсолютному виду

Если пути некорректны, проверьте настройку `workspaceFolder`.
