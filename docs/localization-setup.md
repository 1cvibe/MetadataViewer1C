# Настройка локализации расширения

## Языковая политика

**Русский язык используется по умолчанию для всех пользователей.**

Расширение ориентировано на русскоязычную аудиторию разработчиков 1С:Предприятие.

## Структура файлов локализации

1. **`package.nls.json`** - базовый файл (русский, по умолчанию)
2. **`package.nls.ru.json`** - русская локализация (дубликат базового)
3. **`package.nls.en.json`** - английская локализация (для англоязычных пользователей)

## Как работает локализация

**По умолчанию все команды отображаются на русском языке.**

Пользователи, которым нужен английский интерфейс, могут переключить язык:

### Переключение на английский язык

#### Вариант 1: Глобальные настройки Cursor

1. Откройте Command Palette (`Ctrl+Shift+P`)
2. Введите `Configure Display Language`
3. Выберите `English`
4. Перезапустите Cursor

#### Вариант 2: Через файл настроек

Откройте файл настроек Cursor:
- Windows: `%APPDATA%\Cursor\User\settings.json`
- Linux: `~/.config/Cursor/User/settings.json`
- macOS: `~/Library/Application Support/Cursor/User/settings.json`

Добавьте:

```json
{
  "locale": "en"
}
```

## Проверка текущей локали

Выполните в терминале:

```powershell
$env:VSCODE_NLS_CONFIG
```

Вы должны увидеть:

```json
{
  "locale": "ru",
  "resolvedLanguage": "ru",
  ...
}
```

## После изменения локали

**Обязательно перезапустите Cursor полностью** (не только окно, а весь процесс).

## Проверка работы локализации

По умолчанию (без дополнительных настроек):

1. Откройте Command Palette (`Ctrl+Shift+P`)
2. Найдите команду `BSL: Редактировать запрос`
3. Откройте контекстное меню в BSL файле - все команды на русском языке

При переключении на английский (`"locale": "en"`):

1. Команды отображаются как `BSL: Edit query`
2. Все меню переключаются на английский

## Структура файлов локализации

Все команды в `package.json` используют ссылки вида:

```json
{
  "command": "metadataViewer.editQueryFromEditor",
  "title": "%1c-metadata-viewer.editQueryFromEditor.title%"
}
```

Переводы находятся в:

- `package.nls.json` - **русский (по умолчанию)**
- `package.nls.ru.json` - русский (дубликат)
- `package.nls.en.json` - английский

## Отладка

Если локализация не работает:

1. Проверьте, что файлы `package.nls.*.json` находятся в корне расширения
2. Убедитесь, что Cursor полностью перезапущен
3. Проверьте переменную окружения `VSCODE_NLS_CONFIG`
4. Убедитесь, что в `.vscodeignore` нет исключения для `*.nls.*` файлов
