# Сохранение структуры XML при редактировании метаданных

## Описание проблемы

При редактировании метаданных объектов через редактор `edit metadata` и последующем сохранении изменений структура XML файла изменялась:
- Свойства, которые были **атрибутами** в исходном XML, становились **элементами**
- Свойства, которые были **элементами**, могли становиться **атрибутами**

Это происходило из-за того, что:
1. Парсер `fast-xml-parser` при `attributeNamePrefix: ""` не различает атрибуты и элементы - оба сохраняются как простые свойства объекта
2. Функция `parseProperties` преобразует сложные структуры в простые значения, теряя информацию о структуре
3. Билдер `fast-xml-builder` самостоятельно решает, создавать атрибут или элемент, основываясь на типе значения

## Решение

### Архитектура

Решение основано на сохранении исходного объекта из парсера (`_raw`) вместе с нормализованными данными. При сохранении используется исходная структура для определения формата каждого свойства (атрибут или элемент).

### Компоненты решения

#### 1. Расширение интерфейсов

Добавлено поле `_raw` в интерфейсы:
- `ParsedMetadataObject._raw` - исходная структура объекта из парсера
- `ParsedAttribute._raw` - исходная структура реквизита
- `ParsedTabularSection._raw` - исходная структура табличной части

#### 2. Модификация парсера

Парсер (`src/xmlParsers/metadataParser.ts`) теперь сохраняет исходные объекты:
- В `parseMetadataXml`: сохраняет `objNode` в `_raw`
- В `parseAttributes`: сохраняет исходный объект реквизита в `_raw`
- В `parseTabularSections`: сохраняет исходный объект табличной части в `_raw`

#### 3. Утилита восстановления структуры

Создан файл `src/utils/xmlStructureUtils.ts` с функциями:
- `detectStructureType()` - определяет тип свойства (атрибут/элемент) на основе исходного объекта
- `restorePropertiesStructure()` - восстанавливает структуру Properties с сохранением формата
- `restoreAttributeStructure()` - восстанавливает структуру реквизита
- `restoreTabularSectionStructure()` - восстанавливает структуру табличной части

#### 4. Модификация логики сохранения

Метод `handleSave` в `src/panels/MetadataPanel.ts` теперь:
- Использует `obj._raw` для получения исходной структуры
- Вызывает функции восстановления структуры вместо прямого создания элементов
- Сохраняет формат неизмененных свойств из исходного XML

### Алгоритм работы

1. **Парсинг XML**:
   - Парсер создает нормализованные данные для отображения в редакторе
   - Одновременно сохраняет исходный объект из парсера в `_raw`

2. **Редактирование**:
   - Пользователь редактирует данные через веб-интерфейс
   - Изменения сохраняются в нормализованном формате

3. **Сохранение**:
   - Используется исходная структура из `_raw` для определения формата каждого свойства
   - Измененные свойства обновляются, но сохраняют свой формат (атрибут/элемент)
   - Неизмененные свойства остаются в исходном формате

### Примеры использования

#### Пример 1: Properties с атрибутами

**Исходный XML:**
```xml
<Properties Name="Test" Comment="Comment" UseStandardCommands="true"></Properties>
```

**После парсинга:**
```javascript
{
  properties: { Name: "Test", Comment: "Comment", UseStandardCommands: "true" },
  _raw: {
    Properties: { Name: "Test", Comment: "Comment", UseStandardCommands: "true" }
  }
}
```

**После сохранения:**
```xml
<Properties Name="Test" Comment="Comment" UseStandardCommands="true"></Properties>
```
✅ Структура сохранена

#### Пример 2: Properties с элементами

**Исходный XML:**
```xml
<Properties>
  <Name>Test</Name>
  <Comment>Comment</Comment>
</Properties>
```

**После парсинга:**
```javascript
{
  properties: { Name: "Test", Comment: "Comment" },
  _raw: {
    Properties: {
      Name: { "#text": "Test" },
      Comment: { "#text": "Comment" }
    }
  }
}
```

**После сохранения:**
```xml
<Properties>
  <Name>Test</Name>
  <Comment>Comment</Comment>
</Properties>
```
✅ Структура сохранена

### Ограничения

1. **Эвристика определения типа**: При `attributeNamePrefix: ""` парсер не различает атрибуты и элементы, поэтому используется эвристика на основе структуры объекта. В большинстве случаев она работает корректно, но могут быть edge cases.

2. **Память**: Сохранение исходных объектов увеличивает использование памяти, но это приемлемо для метаданных.

3. **Новые свойства**: Для новых свойств (которых не было в исходном XML) используется формат по умолчанию - элементы.
