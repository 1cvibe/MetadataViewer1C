# Тесты редактирования запросов в DCS схеме

## Описание

Автоматизированные тесты для проверки функционала редактирования запросов в редакторе схем компоновки данных (DCS).

## Файл теста

`test-dcs-query-editing.js`

## Что тестируется

Тесты проверяют корректность логики автоматического обновления полей при редактировании текста запроса:

### ТЕСТ 1: Добавление нового поля
- **Сценарий**: В существующий запрос добавляется новое поле `Поле3 КАК Поле3`
- **Проверка**: Новое поле должно автоматически появиться в списке полей набора данных
- **Ожидаемый результат**: `['Поле1', 'Поле2', 'Поле3']`

### ТЕСТ 2: Редактирование середины запроса
- **Сценарий**: В середину запроса добавляется комментарий (regex не найдет все поля)
- **Проверка**: Существующие поля НЕ должны удалиться
- **Ожидаемый результат**: `['Поле1', 'Поле2', 'Поле3']` (все поля сохранены)

### ТЕСТ 3: Добавление нескольких полей одновременно
- **Сценарий**: К одному полю добавляются сразу три новых
- **Проверка**: Все новые поля должны добавиться к существующим
- **Ожидаемый результат**: `['Поле1', 'Поле2', 'Поле3', 'Поле4']`

### ТЕСТ 4: Запрос с несколькими пакетами
- **Сценарий**: Запрос содержит несколько пакетов, разделенных `;`, поле добавляется в последний пакет
- **Проверка**: Должно извлекаться только поле из последнего пакета запроса
- **Ожидаемый результат**: `['Поле1', 'Поле2', 'Поле3']`

### ТЕСТ 5: Пустой запрос
- **Сценарий**: Текст запроса полностью удаляется
- **Проверка**: Существующие поля НЕ должны удалиться
- **Ожидаемый результат**: `['Поле1', 'Поле2']` (поля сохранены)

## Технические детали

### Используемые технологии
- **Node.js**: Среда выполнения
- **fast-xml-parser**: Парсинг и сборка XML
- **Regex**: Извлечение имен полей из запросов 1С

### Логика извлечения полей

Функция `extractFieldNamesFromQuery` извлекает имена полей из SQL-подобных запросов 1С:
- Поддерживается русский синтаксис: `... КАК ИмяПоля`
- Поддерживается английский синтаксис: `... AS FieldName`
- При наличии нескольких пакетов (разделенных `;`) анализируется только последний

### Логика обновления полей

Функция `updateDataSetQuery` имитирует логику из `src/webview/components/DcsEditor/DcsEditorApp.tsx`:
1. Извлекает поля из нового текста запроса
2. Сравнивает с существующими полями
3. **Добавляет** только новые поля (не удаляет существующие)
4. Если извлечение не удалось - оставляет поля как есть

## Связанные файлы

- `src/webview/components/DcsEditor/DcsEditorApp.tsx` - реальная реализация логики
- `test-cases/test-dcs-query-editing.js` - этот тест

## Изменения в коде

Тесты созданы для проверки исправления, внесенного в `handleChangeDataSetQuery`:
- **До**: При редактировании запроса все поля удалялись и заменялись на найденные
- **После**: Новые поля добавляются к существующим, старые не удаляются

## Exit коды

- `0` - все тесты пройдены
- `1` - есть непройденные тесты
